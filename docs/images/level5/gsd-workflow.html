Discuss → Plan → Execute → Verify 4단계 phase 구조로 Context Rot 없이 장기 프로젝트를 완성하도록 설계된 Claude Code 시스템.
실전 워크플로 Phase 단위 컨텍스트 분리 — 명령 흐름
1
/gsd-new-project 새 프로젝트 초기화 (초기 1회만)
2
/gsd-discuss-phase 1 /clear 후 Phase 논의 — 요구사항 명확화·구현 결정 토론
3
/gsd-plan-phase 1 구현 계획 수립 — 리서치 + 계획 + 검증
4
/gsd-execute-phase 1 병렬 실행 — 서브에이전트 병렬 구현
5
/gsd-verify-work 1 검증 — 서브에이전트 병렬 구현
6
/gsd-ship 1 PR
7
/gsd-next 다음 Phase
핵심 메커니즘 Phase마다 서브에이전트가 독립 컨텍스트를 가진다. 1 Task = 1 Commit 원자적 커밋으로 Context Rot 누적을 차단.
