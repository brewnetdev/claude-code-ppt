모델이 자가 호출하는 절차 묶음 AI 에이전트에 새로운 기능과 전문성을 부여하는 표준화된 방식
CONCEPT
구성 — SKILL.md 한 파일 = 이름 · 설명 · 허용 도구 · 본문 절차. Slash Command와 Skill은 동일 lazy-load 메커니즘 위에서 동작한다.
기본 구조 SKILL.md 디렉터리 구조 + frontmatter · 본문 절차
STRUCTURE
.claude/skills/
└─ my-skill-name/
  └─ SKILL.md  # name · description · allowed-tools + 본문
관점
Slash Command User-invoke
Skill Model-invoke
호출
사용자가 /명령 입력
모델이 description 매칭으로 자동 호출
발견
사용자가 알아야 함, 단일 작업 발동기
모델이 컨텍스트 보고 판단, 재사용 가능한 작업 모듈
본문
프롬프트 텍스트
마크다운 + frontmatter (allowed-tools 등)
통합신규는 SKILL.md로 작성 권장 — 모델 자가 발견·호출이 가능해 사용자 부담이 준다. Command = 단축키 VS Skill = 자가호출 컨텍스트 메뉴.
주의모델은 description만 보고 호출 여부를 결정한다. "언제 써야 하는지"를 명확히 적지 않으면 절대 호출되지 않는다.
예시 .claude/skills/my-skill-name/SKILL.md
---
name: my-skill-name # /{name}으로 호출되는 식별자
description: 무엇을 하고 언제 사용하는지 명확하게. # 자동 활성화 여부 판단
allowed-tools: Bash(npm run:*), Read, Write # 사용 가능 도구
---
