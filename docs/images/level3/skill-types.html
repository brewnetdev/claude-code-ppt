Skill이 수행할 수 있는 작업 유형 4 TYPES
01
파일 시스템 접근
파일 읽기·쓰기·수정에 대한 구체적인 규칙과 패턴을 정의할 수 있다.
02
명령어 실행
특정 상황에서 실행해야 할 셸 명령어나 스크립트를 정의할 수 있다.
03
웹 검색 및 문서 참조
최신 정보나 공식 문서를 참조해야 하는 상황에 대한 지침을 제공할 수 있다.
04
권한 제어나 보안
allowed-tools 필드를 통해 활성화된 스킬의 도구를 제한할 수 있다.
예시 allowed-tools
# 코드 수정 가능 (명령어 실행 불가)
allowed-tools: Read, Write, Edit, Grep, Glob
# 전체 개발 작업 (테스트 실행 포함-보안상 위험)
allowed-tools: Read, Write, Edit, Grep, Glob, Bash
# 전체 개발 작업 (권장 - 필요한 명령만 허용)
allowed-tools: Bash(npm run:*), Bash(git add:*),
  Bash(git commit:*), Read, Write
TIP  자동 호출 여부 판단 시 Claude는 description을 가장 먼저 읽는다 — 설명이 모호하면 활성화도 모호하다.
사용 가능한 Skill 확인 — 3가지 방법 3 WAYS
1 /skills 통합 조회 권장
개인·프로젝트·설치된 플러그인의 Skill을 한 화면에서 통합 조회 — Claude Code CLI에서 가장 간단
2 자연어 질문
"사용 가능한 Skill이 뭐가 있어?"
Claude가 description을 모아 답변
3 직접 확인 — 4개 경로
# ① 개인 스킬 (사용자 전역)
ls ~/.claude/skills/
# ② 프로젝트 스킬
ls .claude/skills/
# ③ 플러그인 설치 스킬 (누락!)
ls ~/.claude/plugins/
# ④ --add-dir 추가 디렉토리 스킬 (선택적, 누락!)
# claude --add-dir ../shared-tools
