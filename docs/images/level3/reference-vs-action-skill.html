Reference Skill 지식·컨벤션 인용형 — 작업할 때 참조, 도구 호출이 거의 없는 형태
REFERENCE
성격
규칙·컨벤션을 인용해 작업 중 참조
allowed-tools
Read 읽기 중심 — 도구 호출 거의 없음
사용 시점
schema.ts 편집 시, 새 테이블/관계 추가 시
예시 .claude/skills/drizzle-conventions/SKILL.md
name: drizzle-conventions
description: Drizzle ORM 스키마·쿼리 작성 시 항상 참고. 컬럼 명명·인덱스·관계·트랜잭션 규칙.
allowed-tools: Read
## 컬럼 명명 — 테이블 snake_case 단수형, created_at/updated_at 필수
## 인덱스 — Drizzle은 FK를 자동 인덱싱하지 않는다 (누락 시 full table scan)
Action Skill 절차·실행형 — "X 요청이 들어오면 Y 절차로 처리해", 도구 호출까지 묶음
ACTION
성격
절차 + 도구 호출을 하나로 묶어 실행
allowed-tools
Bash Read Edit 실행 권한 포함
사용 시점
스키마 파일 수정 후, 또는 "마이그레이션 만들어" 요청 시
예시 .claude/skills/db-migration/SKILL.md
name: db-migration
allowed-tools: Bash(pnpm:*) Bash(drizzle-kit:*) Read Edit
disable-model-invocation: true # /skill-name 명시 입력 시만
1. drizzle-kit generate로 SQL 생성 2. Read로 검토 (DROP 여부)
3. destructive면 사용자 확인 후 진행 4. push/migrate 실행 5. diff 요약
Reference — 모델이 "어떻게 쓸지"의 기준을 제공한다. 결과물은 코드, 스킬은 규칙집.
Action — 모델이 "무엇을 할지"의 절차를 수행한다. argument-hint로 인자 placeholder 안내.
