PERMISSIONS / SPECTRUM
권한 모드 스펙트럼 — 안전에서 자율까지 6모드
v1.1 기준 4종(default · acceptEdits · dontAsk · bypassPermissions)에 plan · auto를 더한 6모드 · 오른쪽으로 갈수록 자동 허용 범위가 넓어진다
안전 · 매번 승인 자율 · 검사 없음
왼쪽일수록 사람이 더 자주 개입 · 오른쪽일수록 모델이 더 많이 자동 실행
01 default Shift+Tab
파일 읽기만 자유. 편집·Bash·외부 호출은 매번 승인 요청. 시작점으로 쓰고 acceptEdits로 이동 권장.
자동 허용: 파일 읽기만
02 acceptEdits Shift+Tab
파일 읽기·편집 자동 승인. mkdir/touch/mv/cp 등도 작업 디렉토리 내 자동 승인. Bash 실행은 여전히 매번 확인.
상태표시줄: ⏵⏵ accept edits on
03 plan Shift+Tab×2
소스코드 편집(Edit·Write)은 차단. 파일 읽기와 탐색용 Bash는 가능. 코드베이스를 분석·계획하되 변경하지 않음.
상태표시줄: ⏸ plan mode on
04 auto Team/Ent/API
모든 작업을 백그라운드 안전성 검사와 함께 자동 실행. 분류기(classifier)가 요청 의도와 행동 일치를 판단.
시작 플래그: --enable-auto-mode
05 dontAsk CI/CD · Headless
allow 리스트에 없는 모든 도구를 자동 거부. 거부 시 메시지 없이 조용히 차단해 디버깅이 어려움.
자동 허용: 사전 승인된 도구만
06 bypass​Permissions 위험 · 격리 환경 전용
권한 프롬프트를 건너뛰고 완전 자율 실행. 단 .git .claude .vscode .idea .husky 쓰기는 여전히 확인 요청.
시작 플래그: --dangerously-skip-permissions
Shift+Tab 사이클은 default ↔ acceptEdits 전환 · plan은 별도 진입(두 번)
안전 수칙 acceptEdits는 반드시 git checkpoint 후 사용 · deny 룰 권장: Bash(git push *), Bash(rm -rf *)
bypassPermissions는 검사 없이 모든 작업 실행 — 격리 컨테이너 · VM 전용
