1.1.6 GIT CLI · BASIC CYCLE
Git 기본 사이클 — 로컬에서 원격까지의 흐름
clone / init 으로 시작해 add → commit → push 로 내보내고, fetch / pull 로 원격 변경을 다시 받는다.
LOCAL
Working Directory
워킹 디렉토리
파일을 실제로 수정하는 작업 공간. git status 로 매번 먼저 확인.
LOCAL
Staging Area
스테이징 영역
다음 커밋에 담을 변경을 모아두는 대기 공간. 무엇이 바뀌었나 추려낸다.
LOCAL
Local Repository
로컬 저장소 · .git
작업 내용을 로컬에 저장 — 세이브포인트, 체크포인트. 커밋 히스토리가 쌓인다.
git merge · git checkout -b — 브랜치 합치기·이동 (로컬 내부)
REMOTE
Remote
원격 저장소 · origin
GitHub 등 공유 저장소. 협업의 단일 기준점이 된다.
git add
변경 → 스테이징
git commit
스테이징 → 스냅샷
git push
로컬 → 원격
git fetch ◂ Remote 원격의 변경사항을 확인만 함 (로컬에 소스 변경 X) — 안전.
git pull ◂ Remote 원격의 변경사항을 현재 브랜치에 적용 (fetch + merge).
git clone ◂ Remote 원격 저장소를 내 컴퓨터로 복제 — 프로젝트 처음 받기.
로컬 (LOCAL)
원격 (REMOTE)
로컬 내부 명령
전송 방향:add → commit → push·fetch / pull ◂ 원격
git status · git diff · git diff --staged 로 단계마다 확인
