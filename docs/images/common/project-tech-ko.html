# 핵심 기술 무엇을 배우는가 — 의미·맥락
01 SSE (Server-Sent Events)단방향 푸시 · WebSocket 대안 메뉴판처럼 읽기 전용·서버→클라이언트 일방향이면 WebSocket보다 가볍다. 본 프로젝트에서는 "data: 1" 한 글자 트리거만 보낸다.
02 파일 감시 + 디바운스chokidar · FSEvents · 300ms OS 파일 시스템 이벤트로 외부 변경에 자동 반응. Excel 저장 시 중복 이벤트를 debounce 300ms 로 한 번으로 합친다 — 실서비스 안정성의 기초.
03 풀(pull) / 푸시(push) 분리알림 채널 + 데이터 채널 분리 알림은 가볍게 푸시(SSE), 데이터는 요청 시 풀(fetch). 채널이 독립이라 알림 채널은 단순하게 유지되고, 데이터는 항상 파일의 최신 상태를 반영한다.
04 단일 진실원천 (SSoT)Single Source of Truth Excel 직접 편집과 /admin UI 두 채널이 결국 같은 menu.xlsx 한 파일을 바꾼다. 출처가 하나라 동기화·정합성 문제가 사라진다.
05 데이터 / 표현 분리Excel = 데이터 · CSS = 표현 Excel은 데이터, cafe-dark / bistro-light CSS 템플릿은 표현. 코드 수정 없이 운영자가 _설정 시트의 한 셀로 디자인을 바꾼다.
