1 TDD 사이클
/tdd-red 실패 테스트
/tdd-green 최소 구현
/tdd-refactor 중복 제거
핵심각 단계가 별도 커맨드 — 모델이 단계 혼용을 못 한다.
2 자동화 파이프라인
/lint-fix
/typecheck
/test-affected
/commit-msg
핵심CI 전 단계를 슬래시로 묶어 단계별 일관된 사전 점검.
3 컨텍스트 오염 방지
/checkpoint 핵심 결정 dump
/clear
/restore-checkpoint 깨끗한 상태에서 재개
핵심긴 디버그 세션의 누적 오류 차단.
패턴의 공통점 "한 작업 = 한 커맨드"로 행위를 명확히 끊고, 각 커맨드가 자기 책임만 갖는다.
