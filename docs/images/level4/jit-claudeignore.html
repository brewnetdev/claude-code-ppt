왜 JIT 인가JUST-IN-TIME 전체 프로젝트 인덱싱은 비현실적 — n² 어텐션 비용. 지금 작업과 무관한 파일은 노이즈 → Context Rot # 전체 인덱싱 대신 필요한 곳만 적시 로드
적시 컨텍스트 모델이 직접 Glob/Grep/Read로 필요한 곳만 읽게 함 Glob → Grep → Read
경로만 남기는 패턴CLAUDE.md ## 핵심 파일 — 경로 + 한 줄 설명만 적어 진입점을 알린다 - src/server/db/schema.ts - src/lib/markdown.ts - src/components/editor/CodeMirror.tsx - drizzle/
.claudeignore노이즈 차단 인덱싱에서 제외할 디렉터리·파일 패턴을 선언해 Glob 노이즈를 줄인다 node_modules/  dist/  .next/  coverage/ *.min.js  *.csv  *.log  public/uploads/ drizzle/meta/_journal.json  src/generated/
효과 인덱싱 토큰과 Glob 노이즈가 적용 후 크게 줄어든다 인덱싱 토큰  ~85K → ~22K Glob 노이즈  다수 → 소스만
정리 CLAUDE.md는 경로만 남겨 진입점을 알리고, .claudeignore로 노이즈를 차단하면 모델이 적시에 필요한 파일만 읽는다.
