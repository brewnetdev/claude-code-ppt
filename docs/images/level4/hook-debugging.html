CAVEATS 훅 작성 시 주의사항
사용자 권한 실행 위험
훅은 사용자 권한으로 실행. 시스템·DB·자격증명에 영향을 주는 명령을 무심코 적지 말 것 — 훅은 자동 실행
jq 미설치 주의 위험
CI·서버에서 brew·apt로 jq 설치 필수 — 없으면 훅이 조용히 실패
exit 1 = 사용자만 전달
비차단(exit 1) 메시지는 사용자만 봄. 모델에게 사유를 전달하려면 exit 2 + stderr 필수
매처 누락 = 전체 위험
PreToolUse에 matcher 안 적으면 Read·Grep까지 가로챔 → 성능 저하
DEBUGGING
디버깅
claude --debug로 훅 발화·exit code·stderr 추적. 훅 자체에 set -x + 로그 파일 저장 권장
claude --debug # 훅 발화·exit code·stderr 추적

# 훅 스크립트 상단
set -x
exec 2>> "$HOME/hook.log" # 로그 저장
설계 원칙 훅은 "1초 안에 끝나야" 한다
느린 훅은 모든 도구 호출을 지연시킨다. 무거운 검증은 PostToolUse + 비동기로 처리한다.
