Red→Green→Refactor
Red 1
실패하는 테스트 먼저
구현 없이 테스트만 작성, 실패를 먼저 확인한다
↓
Green 2
통과하는 최소 구현
테스트를 통과시키는 최소한의 코드만 작성한다
↓
Refactor 3
구조 개선
테스트 통과를 유지하며 코드를 정리한다
↺통과 상태를 유지하며 반복
단계별 규칙 — 명시적으로 TDD 사이클 요청하기
Red 실패하는 테스트만
1구현 코드는 절대 작성하지 않음
2테스트 파일만 생성 또는 수정
3테스트 실행하여 실패 확인
Green 최소한의 코드로
1테스트를 통과시키는 최소한의 코드만 작성
2하드코딩도 허용 (다음 테스트가 일반화를 강제)
3"미래를 위한" 코드 금지
Refactor 통과 유지하며 정리
1새로운 기능 추가 금지, 테스트 수정 금지
2각 리팩터링 후 테스트 실행
3중복 제거 · 명확한 이름 · 메서드 추출
