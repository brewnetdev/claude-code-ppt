Non-deterministic — 같은 입력, 다른 출력
Deterministic 프로그래밍과 달리 같은 요청에도 다른 결과가 나온다.
DETERMINISTIC 같은 입력 → 항상 같은 출력 (일반 프로그래밍)
NON-DETERMINISTIC 같은 입력 → 매번 다른 출력 (AI 생성)
대응
테스트 = 결정적 검문소
AI가 어떤 방식으로 구현하든 테스트를 통과하면 요구사항 충족. 결과물의 '동일함'이 아니라 '동작함'이 성공의 기준이다.
다른 출력 비결정성이 드러나는 3가지 양상
1
미묘하게 다른 구현
정규식 / 조건문 / 외부 라이브러리 — 매번 다른 접근, 같은 입력을 해도 매번 다른 출력이 나온다.
2
예상치 못한 변경
관련 없는 부분까지 변경, 빌드 성공 주장이 거짓일 수도 있다.
3
치팅 징후가장 위험
테스트를 통과시키기 위해 테스트 자체를 비활성화·삭제 — 테스트를 수정하는 게 아니라 Skip/Delete 한 뒤 통과한 것처럼 행동한다.
