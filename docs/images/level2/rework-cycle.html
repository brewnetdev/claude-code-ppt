ANTI-PATTERN 설계 없는 개발 — 재작업 사이클
1주 만에 전면 재작성으로 이어지는 전형적 흐름
1일차 "TODO 웹 애플리케이션 만들어줘"
→ 개인용 CRUD 생성
↓
3일차 "팀 공유 기능 추가해줘"
→ 인증/권한 시스템 급조
↓
7일차 "SaaS로 서비스하고 싶어"
→ 멀티테넌시 구조 전면 재작성
↓
14일차 "온프레미스 설치 옵션도 필요해"
→ 환경 설정 시스템 재설계
↻ 요청 → 코드 → 불일치 → 재작업 반복
VS
DESIGN-FIRST 설계 후 1회 통과
구현 전 선행되어야 할 3가지를 먼저 정의한다
1 요구사항 분석
무엇을 만들 것인가 — 핵심·비기능 요구사항 수립
2 아키텍처 결정
어떤 기술·구조로 만들 것인가 — 빌드/배포까지 영향
3 인터페이스 정의
기능 정의, API 규격, 데이터 흐름 사전 정의
↓
1회 통과 — 요청과 결과 일치
Garbage In, Garbage Out 입력의 품질 = 출력의 품질. 명세 없는 요청은 AI가 임의로 결정하게 만든다.
