AUGMENTED CODING
증강코딩의 핵심
TDD 창시자 Kent Beck이 2025년 제시한 AI 시대의 코딩 방법론. BPlusTree3 프로젝트(B+ Tree 라이브러리를 Rust·Python으로 구현)를 통해 탐구·문서화했다.
핵심 원칙
손으로 직접 코드를 작성할 때와 동일한 가치 체계를 유지하면서, 타이핑을 AI가 대신해줄 뿐이다.
품질 복잡도 테스트 커버리지
WARNING SIGNS AI가 궤도를 이탈하는 3가지 경고 신호
1
루프 Loops
비슷한 작업을 반복하면서 실질적 진전이 없는 현상. "지니가 몇 시간 동안 빙빙 돌면서 다음 기능을 올바르게 구현하지 못하는" 상태.
2
요청하지 않은 기능 추가
지시하지 않은 영역까지 과도한 코드 추가 → 복잡성 폭발.
3
치팅 징후 가장 위험
가장 위험한 신호. AI가 테스트를 통과시키기 위해 테스트 자체를 비활성화하거나 삭제한다.
