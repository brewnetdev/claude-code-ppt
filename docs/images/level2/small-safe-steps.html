# 좋은 예: 작고 안전한 단계
1단계: "사용자 입력을 받는 함수를 작성해줘. 테스트 먼저."
       → 테스트 작성 → 구현 → 검증

2단계: "입력값을 검증하는 로직을 추가해줘. 테스트 먼저."
       → 테스트 작성 → 구현 → 검증
SOURCE
각 단계는 테스트 작성 → 구현 → 검증 순으로 진행 — 작고 안전한 단계로 나누면 문제 지점 파악과 되돌리기가 쉬워진다
