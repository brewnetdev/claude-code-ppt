00이 문서가 다루는 것 · 전체 흐름

터미널을 처음 꾸미는 개발자가 복사–붙여넣기만으로 보기 좋고 빠른 zsh 환경을 완성하도록 만든 단계별 가이드입니다. 각 단계는 OS별 명령을 모두 제공합니다.

전체 설치 흐름

01zsh란 무엇인가

zsh(Z shell)는 bash와 호환되면서 자동완성·테마·플러그인 확장성이 훨씬 강력한 셸입니다. macOS는 Catalina(10.15)부터 기본 셸이 zsh이며, 대부분의 개발 도구·튜토리얼이 zsh를 전제로 합니다.

구분bashzsh
자동완성기본적강력(퍼지·대소문자 무시·메뉴 선택)
테마/플러그인제한적oh-my-zsh 등 풍부한 생태계
macOS 기본과거 기본현재 기본

02사전 확인

# 현재 셸 확인
echo $SHELL

# zsh 설치 여부 / 버전 확인
zsh --version          # zsh 5.x 이상이면 OK
Git이 먼저 필요합니다

Git이 없으면 테마·플러그인 설치(git clone)가 안 됩니다. git --version으로 확인하고, 없으면 macOS는 xcode-select --install, Ubuntu는 sudo apt install -y git로 설치하세요.

03zsh 설치 (OS별)

3.1 macOS

macOS는 zsh가 기본 내장이라 설치 불필요입니다. 버전만 확인합니다.

zsh --version

최신 버전을 원하면 Homebrew로 업그레이드합니다(선택).

brew install zsh
Homebrew가 없다면 먼저 설치

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

3.2 Ubuntu / WSL2

sudo apt update && sudo apt install -y zsh
zsh --version

3.3 Windows

Windows의 PowerShell·CMD에는 zsh가 없습니다. zsh는 WSL2(Ubuntu) 위에서 사용하는 것을 권장합니다.

# PowerShell(관리자)에서 WSL2 + Ubuntu 설치
wsl --install

설치 후 Ubuntu 터미널에서 3.2 Ubuntu / WSL2 절차를 그대로 따릅니다.

[ 스크린샷 영역 ]
WSL2 Ubuntu 첫 실행 화면

04기본 셸을 zsh로 변경

# 현재 시스템의 zsh 경로 확인
which zsh              # 예: /bin/zsh 또는 /usr/bin/zsh

# 기본 셸 변경
chsh -s $(which zsh)

변경 후 터미널을 완전히 종료하고 다시 열거나 로그아웃/로그인합니다. 다시 echo $SHELL이 zsh 경로를 가리키면 성공입니다.

macOS 참고

macOS는 이미 zsh가 기본이라 이 단계가 생략될 수 있습니다. 결과가 /bin/zsh면 그대로 진행하세요.

05oh-my-zsh 설치선택 · 권장

oh-my-zsh는 테마·플러그인을 손쉽게 관리하는 프레임워크입니다.

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

설치가 끝나면 홈 디렉터리에 ~/.oh-my-zsh/가 생성되고 ~/.zshrc가 oh-my-zsh용으로 교체됩니다.

[ 스크린샷 영역 ]
oh-my-zsh 설치 완료 직후 터미널 화면
성능 참고

oh-my-zsh는 편리하지만 플러그인이 많아질수록 셸 시작이 느려집니다(수백 ms). 속도가 중요하면 7.4의 Starship을 고려하세요.

06Nerd Font 설치아이콘 테마 시 필수

agnoster·Powerlevel10k 등 powerline/아이콘 글리프를 쓰는 테마는 Nerd Font가 있어야 깨지지 않습니다. 미니멀 테마나 Starship 기본 사용 시에는 없어도 됩니다.

# macOS (Homebrew)
brew install --cask font-meslo-lg-nerd-font
# Ubuntu / WSL2 — 수동 설치 예시 (Meslo)
mkdir -p ~/.local/share/fonts && cd ~/.local/share/fonts
# nerdfonts.com 의 Releases에서 원하는 폰트(zip)를 받아 압축 해제 후:
fc-cache -fv

설치 후 터미널 앱의 폰트 설정을 Nerd Font(예: MesloLGS NF)로 바꿔야 적용됩니다.

macOS TerminalSettings → Profiles → Text → Change Font
Windows Terminal(WSL)Settings → 프로필 → 모양 → 글꼴
VS Code 내장 터미널terminal.integrated.fontFamily 설정

07테마 설치 & 설정

~/.zshrc의 ZSH_THEME="..." 한 줄로 테마를 바꿉니다. 변경 후 항상 source ~/.zshrc(또는 터미널 재시작).

테마 한눈 비교

테마형태특징Nerd Font비고
robbyrussellomz 내장omz 기본값, 초경량불필요입문 무난
agnosteromz 내장클래식 powerline 세그먼트필요깔끔한 git 표시
Powerlevel10komz 외부가장 인기, instant prompt, 설정 마법사권장2026 유지보수 모드(정상 동작)

7.1 robbyrussell / agnoster (내장 — 설치 불필요)

# ~/.zshrc 편집
ZSH_THEME="robbyrussell"   # 또는 "agnoster"
source ~/.zshrc

7.2 Powerlevel10k (가장 인기)

git clone --depth=1 https://github.com/romkatv/powerlevel10k.git \
  ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/powerlevel10k
# ~/.zshrc
ZSH_THEME="powerlevel10k/powerlevel10k"
source ~/.zshrc
p10k configure        # 대화형 설정 마법사 (아이콘/색/레이아웃 선택)
[ 스크린샷 영역 ]
p10k configure 마법사 첫 화면 (다이아몬드/아이콘 인식 질문)
마법사 재실행

설정을 다시 잡고 싶으면 언제든 p10k configure를 재실행합니다. 설정은 ~/.p10k.zsh에 저장됩니다.

7.3 Spaceship

git clone https://github.com/spaceship-prompt/spaceship-prompt.git \
  "${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/spaceship-prompt" --depth=1
ln -s "${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/spaceship-prompt/spaceship.zsh-theme" \
  "${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/spaceship.zsh-theme"
# ~/.zshrc
ZSH_THEME="spaceship"

7.4 Starship (크로스셸 · 2026 신규 권장)

Starship은 oh-my-zsh의 프롬프트만 대체합니다. 플러그인은 oh-my-zsh를 그대로 쓰면서 프롬프트만 Starship으로 둘 수 있습니다.

curl -sS https://starship.rs/install.sh | sh
# ~/.zshrc — oh-my-zsh 테마와 충돌 방지를 위해 ZSH_THEME는 비워둔다
ZSH_THEME=""

# 파일 맨 아래에 추가
eval "$(starship init zsh)"
source ~/.zshrc

설정 파일은 ~/.config/starship.toml이며, starship preset 명령으로 프리셋을 적용할 수 있습니다.

mkdir -p ~/.config
starship preset gruvbox-rainbow -o ~/.config/starship.toml   # 예시 프리셋
폰트 참고

Starship은 특별한 폰트 없이도 동작하지만, 아이콘을 쓰려면 Nerd Font를 권장합니다.

08필수 플러그인 (강력 추천)

자동완성 제안과 명령어 색상 강조는 생산성을 크게 올립니다. oh-my-zsh의 custom 플러그인으로 설치합니다.

# 1) 입력 중 회색으로 명령 제안
git clone https://github.com/zsh-users/zsh-autosuggestions \
  ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

# 2) 유효/무효 명령을 색으로 구분
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git \
  ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

# 3) 추가 자동완성 정의
git clone https://github.com/zsh-users/zsh-completions.git \
  ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/plugins/zsh-completions

# 4) 위/아래 화살표로 히스토리 부분검색
git clone https://github.com/zsh-users/zsh-history-substring-search.git \
  ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/plugins/zsh-history-substring-search

~/.zshrc의 plugins=(...)에 추가합니다. 순서 중요 — zsh-syntax-highlighting은 항상 마지막에 둡니다.

plugins=(
  git
  zsh-autosuggestions
  zsh-completions
  zsh-history-substring-search
  zsh-syntax-highlighting   # 반드시 마지막
)
source ~/.zshrc

09완성된 ~/.zshrc 예시 (Powerlevel10k 기준)

# ===== oh-my-zsh =====
export ZSH="$HOME/.oh-my-zsh"

ZSH_THEME="powerlevel10k/powerlevel10k"

plugins=(
  git
  zsh-autosuggestions
  zsh-completions
  zsh-history-substring-search
  zsh-syntax-highlighting   # 반드시 마지막
)

source $ZSH/oh-my-zsh.sh

# ===== Powerlevel10k 설정 로드 =====
[[ -f ~/.p10k.zsh ]] && source ~/.p10k.zsh

# ===== 개인 alias =====
alias ll="ls -alh"
alias gs="git status"
alias gd="git diff"
alias ..="cd .."

# ===== 히스토리 =====
HISTSIZE=10000
SAVEHIST=10000
setopt HIST_IGNORE_DUPS        # 중복 명령 저장 안 함
setopt SHARE_HISTORY           # 여러 탭 간 히스토리 공유

# history-substring-search 키 바인딩 (↑ ↓)
bindkey '^[[A' history-substring-search-up
bindkey '^[[B' history-substring-search-down
Starship을 쓸 경우

ZSH_THEME=""로 바꾸고, ~/.p10k.zsh 줄 대신 파일 끝에 eval "$(starship init zsh)"를 둡니다.

112026 동향 메모 (강의 보충용)

  • Powerlevel10k는 유지보수 모드입니다. 신규 기능 개발이 멈췄고 일부 버그는 미수정 상태지만, 아카이브가 아니며 기존 사용자에게는 완전히 정상 동작합니다(최신 푸시 2026-03-14). 기존 환경은 그대로 써도 무방합니다.
  • 신규 셋업이라면 Starship이 현재 표준에 가깝습니다. Rust 컴파일 바이너리라 렌더가 빠르고, bash·zsh·fish·PowerShell 등에서 동일한 설정 파일 하나로 동작합니다. 셸을 바꿔도 프롬프트 설정이 그대로 따라옵니다.
  • 정리: 빠르고 미래지향 → Starship, 풍부한 기능·익숙함 → Powerlevel10k, 가볍게 시작 → robbyrussell / agnoster.
