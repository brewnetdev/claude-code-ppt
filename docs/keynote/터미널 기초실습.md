## 2. 첫 번째 단어 — `pwd` : "나 지금 어디 있어?" (약 2분)

컴퓨터 안에서 우리는 항상 어떤 폴더 '안에' 서 있습니다. 내 위치가 어디냐 이걸 출력하는거에요 

💻 **명령어**
```bash
pwd        # print working directory (현재 위치 출력)
```

🖥️ **예상 출력**
```
/Users/your-name
```

🎙️ **대사 (출력 보고)**
"이게 지금 여러분이 서 있는 위치예요. `/Users/유저 계정명 `. 이걸 '홈 디렉토리'이라고 부를게요. 컴퓨터를 켜면 항상 여기서 있고요, 프로그램 설치나 디렉토리 위치 이동하다가 길 잃어버리면 일단 cd ~ 표시로 언제든지 홈으로 이동합니다. 
그 다음 cd 한칸 띄고 ..을 누르면 현재 위치 바로 위 디렉토리로 갑니다. 
cd music으로 이동했다가 다시 cd ..으로 이동하면 pwd 이렇게 되는거죠 cd는 change directory 입니다. 


## 3. 두 번째 단어 — `ls` : "여기 뭐가 있어?" (약 2분)

🎙️ **대사**
"두 번째 단어 `ls`. 두 글자예요. '지금 이 폴더 안에 뭐가 들어있어?'라고 물어보는 거예요. 

💻 **명령어**
```bash
ls -a -l 
```
ls -a -l  조합을 붙여서 쓸수있습니다. a는 all, l은 long 길게 다 보여줘 이고 a옵션 플래그로 인해 시스템 안에 숨김 폴더 까지 확인할 수 있어요. 파인더나 탐색기로 폴더를 열면 .claude, .env 이런 숨김 폴더들이 안나오는데 -a 옵션이 이 폴더 까지 다 보여주도록 합니다. ls -al로 조합해서 쓸수있다고 이미 설명 드렸습니다. 

🖥️ **예상 출력 (예시 — `-l`이라 한 줄에 하나씩, `-a`라 숨김 항목 `.`·`..`·`.zshrc`까지 보임)**
```

drwxr-xr-x   5 root       admin   160  3  1 09:00 ..
-rw-r--r--   1 your-name  staff   220  3  5 11:10 .zshrc
drwxr-xr-x   4 your-name  staff   128  3  9 17:30 Desktop
drwxr-xr-x   3 your-name  staff    96  3  8 10:05 Documents
drwxr-xr-x   2 your-name  staff    64  3  2 08:40 Downloads
```

## 5. 네 번째 단어 — `mkdir` : "새 폴더 만들기" (약 2분)

다음은 `mkdir`. '새 폴더 만들어'라는 뜻이에요. make directory의 줄임말입니다. 우리 이제 클로드 코드로 작업할 첫 프로젝트 방을 직접 하나 만들어볼게요. 먼저 바탕화면으로 가서—"

💻 **명령어**
```bash
cd ~/music

```
그리고 -p 옵션을 쓰면 중간 부모 폴더까지 한 번에 만들 수 있습니다(깊은 경로 통째로).

💻 **명령어 (mkdir -p)**
```bash

```

## 6. 다섯 번째 — 경로 개념 : "절대 주소 vs 상대 주소" (약 2분)

🎙️ **대사**
"다섯 번째는 단어가 아니라 개념이에요. '경로'. 쉽게 말하면 폴더의 주소예요. 주소에는 두 종류가 있어요.

하나는 **절대 주소**. 우리나라 어디서든 통하는 도로명 주소 같은 거예요. 항상 `/`로 시작합니다."

💻 **예시 (절대 경로)**
```bash
cd /Users/your-name/Desktop/my-first-claude
```

🎙️ **대사**
"또 하나는 **상대 주소**. '여기서 옆 방'처럼, 지금 내 위치 기준으로 말하는 거예요."

💻 **예시 (상대 경로)**
```bash
cd test 
혹은 cd ./test 해도 동일합니다. .이 현재 위치를 말하는 거에요. 이거 나중에 html이나 웹 페이지 만들때 경로 절대경로인지, 상대경로인지 이걸 파악하는데 중요한 개념입니다. ../는 한칸 위 폴더, ./는 현재 위치 폴더나 파일 /는 절대 경로 
```

그리고 touch로 파일을 만들고 cat으로 파일을 볼수있습니다. 

💻 **명령어 (touch · cat)**
```bash
touch memo.txt        # 빈 파일 memo.txt 생성 (이미 있으면 내용은 그대로, 수정 시각만 갱신)
cat memo.txt          # 파일 내용을 터미널에 그대로 출력
cat -n memo.txt       # -n (number) : 줄마다 번호를 붙여서 출력
```
---

가장 위험한 명령어는 rm 입니다. 폴더나 파일 삭제할 때 필요한건데요

💻 **명령어 (rm · 옵션 플래그)**
```bash
rm memo.txt           # 파일 1개 삭제 — 휴지통을 안 거치고 바로 사라짐(복구 어려움)
rm -i memo.txt        # -i (interactive) : "지울까요? y/n" 매번 확인 — 안전장치
rm -r test            # -r (recursive)   : 폴더 + 그 안의 모든 파일·하위폴더까지 통째로 삭제
rm -rf test           # -r + -f (force)  : 확인·경고 없이 강제로 폴더째 삭제 — 가장 위험
```
클로드 코드 쓰다가 rm 관련 경고가 나왔다..? 그럼 주의깊게 살펴봐야 합니다.
나도 모르게 작업 폴더를 날려버릴 수 가 있어요. 
오늘 터미널 기초를 배우는 이유중에 이게 가장 중요합니다.
물론 깃 사용이나 라이브러리 설치시, 에러 로그 긁어서 복사붙여넣기 다 중요한데 이게 제일 중요해요.

또 파일을 만들면서 특정한 문자열을 넣거나 열어서 저장할 수 있어요. 실습해보도록 하겠습니다.

💻 **명령어 (echo 리다이렉션 · 편집기)**
```bash
echo "hello" > memo.txt     # >  : 문자열을 파일에 새로 씀 (기존 내용을 덮어씀)
echo "world" >> memo.txt    # >> : 파일 끝에 한 줄 추가 (이어쓰기, 기존 내용 유지)
cat memo.txt                # 결과 확인 → hello / world 두 줄
nano memo.txt               # 편집기로 열어서 직접 수정 → Ctrl+O 저장, Ctrl+X 종료
```

마지막 예제는 점유하고 있는 포트를 알아내서 kill로 프로세스를 해제하는, 우리가 보통 프로세스 죽인다라고 표현하는데 
lsof 라는 걸 한번 실습 해보도록 하겠습니다. 
서버를 런 한뒤에 다른 탭에서 또 런을 하는 경우 예시에요 

💻 **명령어 (lsof · kill · 옵션 플래그)**
```bash
npm run dev               # 서버 실행 → 포트 3000 점유 (예시)

# ↓ 다른 탭에서 또 실행하면 이런 에러가 납니다
# Error: listen EADDRINUSE: address already in use :::3000

lsof -i :3000             # -i :3000 → 3000번 포트를 점유한 프로세스 조회 (lsof = list open files)
# COMMAND   PID   USER   ...   NAME
# node    12345   you    ...   *:3000 (LISTEN)   ← PID 12345 가 점유 중

kill 12345                # 해당 PID 종료 — 정상 종료 신호(SIGTERM) 전송
kill -9 12345             # -9 (SIGKILL) : 정상 종료가 안 될 때 강제 종료 (최후의 수단)
```

한 줄로 "찾기 + 종료"를 한 번에 끝내는 조합도 있어요.
```bash
kill -9 $(lsof -t -i:3000)   # -t (terse) : PID만 출력 → 그 PID를 바로 kill
```

그러니까 기존 프로젝트가 제대로 shutdown이 안되서 이미 포트가 점유되고 있는 상태에서 이미 점유하고 있는 포트를 찾아 
정리 해두는, 이건 생각보다 자주 접하기 때문에 서버를 기동하는 시작 스크립트에 아에 클로드 코드를 통해서 미리 포트를 셧다운 하고 스타트 해두도록 설정해달라고 요청하면 근본적으로 해결이 됩니다.

제가 커뮤니티 사이트에 업로드한 터미널을 보시면 거기에 나노 에디터에 대해서 정리도 해놨으니까 vi, nano 에디터도 실습을 같이 해보시면
도움이 됩니다.
