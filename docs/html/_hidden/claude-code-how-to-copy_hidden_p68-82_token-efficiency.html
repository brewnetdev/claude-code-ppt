탐색 비용을 줄이는 두 도구 — 정밀 탐색(LSP)과 구조 사전 주입(codesight)을 함께.
🎯 WHY
왜 배우나

새 세션마다 AI가 grep으로 코드를 헤매며 수만 토큰을 낭비한다. 같은 비용을 매번 다시 치른다.

📚 WHAT
무엇을 배우나
  • ▸ LSP — 심볼 단위 정밀 탐색
  • ▸ codesight — 구조 맵 사전 주입
  • ▸ 설치 · 사용법 · 실측 벤치마크
  • ▸ 언제 무엇을 쓸지 의사결정
🛠️ HOW
어떻게 배우나

grep의 한계 이해 → LSP 연결 → codesight 구조 주입 → 실측 수치로 검증 → 상황별 조합 선택.

⭐
CHECKPOINT · 핵심 한 줄
토큰 절감은 모델을 바꾸는 게 아니라 AI에게 읽힐 입력의 형태와 양을 바꾸는 데서 나온다 — “Structure is mechanism”.

토큰 = AI가 한 번에 처리하는 글자 묶음(입력 = AI에게 읽히는 양, 출력 = AI가 쓰는 양). 같은 질문에 AI에게 얼마나 읽히느냐가 돈·속도·정확도를 가른다.

LSP 정밀심볼 단위 정밀 탐색정의 찾기 43배 · 참조 찾기 2.1배 절감"42쪽 14번째 줄" — 위치 좌표만 건넴 (노이즈 0)
codesight 구조프로젝트 구조 사전 파악세션당 6.8배(≈27,915토큰) 절감책 맨 앞에 끼운 요약 지도 한 장 — 매번 뒤지는 일을 없앰
🤝 셋은 경쟁이 아니라 상호 보완 — codesight로 세션 시작 시 "지도"를 깔고, LSP로 "정밀 좌표"를 찍는다.
💡 단, 입·출력 구조는 토큰 효율화의 출발점일 뿐 — 실제 비용은 누적(세션)·캐시·모델 단가 세 축이 함께 좌우한다.

LSP란 무엇인가
Language Server Protocol — 편집기와 언어 서버 사이의 통신 표준. 쉽게: 도서관 "색인 카드" — 본문 대신 "그 단어는 몇 쪽 몇 줄"이라는 위치만 알려준다.

편집기와 언어 서버 간 JSON-RPC 기반 통신 규약. Microsoft가 2016년 오픈 표준으로 공개했고, 최신 안정 버전은 3.17.

LSP 자체는 코드를 분석하지 않는다 — 실제 분석은 각 언어의 언어 서버(vtsls · pyright · rust-analyzer)가 수행하고, LSP는 그 결과를 전달한다.

Claude Code는 기본적으로 grep(텍스트 검색)으로 코드를 탐색한다. 파일이 많아지면 느려지고, 주석·문자열 속 동일 텍스트까지 섞여 정확도가 떨어진다. LSP 연결 시 IDE 수준의 코드 인텔리전스를 얻는다.

goToDefinition-심볼이 선언된 위치로 점프 findReferences-전체 위치 반환 getDiagnostics-언어서버 매칭 조회(에러/경고) hover-타입정보 documentSymbol-파일 구조 아웃 라인 반환 (클래스, 함수, 메서드, 변수 목록)
효과적인 상황
🔧 리팩토링 — findReferences로 실제 참조만 정확히 파악 (누락·오탐 방지)
⚡ 대규모 코드베이스 — 정의 찾기가 밀리초 단위
✅ 수정 직후 검증 — getDiagnostics로 빌드 없이 타입 에러 즉시 확인
⚠️ 파일 20개 미만 소규모 프로젝트나 단순 파일 생성 작업이면 LSP 없이도 충분.
💡 https://microsoft.github.io/language-server-protocol/specifications/lsp/3.17/specification

CH.3 · PART 2 · 사용법
설치 · 핵심 명령어 · 연동
설치 불필요 — npx로 즉시 실행. Claude Code 연동은 3단계.
핵심 명령어
# 스캔 → .codesight/ 생성
npx codesight
npx codesight ./my-project

# AI 설정 파일 (시작 시 자동 로딩)
npx codesight --init
npx codesight --profile claude-code

# 분석 · 자동화
npx codesight --mcp      # MCP 서버 (13개 도구)
npx codesight --hook     # commit마다 재생성
npx codesight --watch    # 변경 감지 재스캔
npx codesight --blast src/db.ts
npx codesight --benchmark
Claude Code 연동 (권장)
# 1) CLAUDE.md 생성 (최초 1회)
npx codesight --init

# 2) .claude/settings.json 에 MCP 등록
{
  "mcpServers": {
    "codesight": {
      "command": "npx",
      "args": ["codesight", "--mcp"]
    }
  }
}

# 3) commit마다 자동 갱신
npx codesight --hook
⚡ --init은 정적 파일 1회 주입, --mcp는 AI가 필요한 것만 실시간 쿼리 → 토큰 효율 최고. 세션 캐싱으로 첫 호출만 스캔.
