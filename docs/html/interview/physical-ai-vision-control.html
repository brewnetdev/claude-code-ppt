00Physical AI란? · 왜 지금인가난이도 쉬움

Physical AI(피지컬 AI)는 embodied AI(체화 AI)와 같은 말로, 화면 속 텍스트·이미지에 머무는 AI가 아니라 카메라·센서로 물리 세계를 인지(perception)하고, 로봇 몸체(actuator)로 행동(action)해 세계의 상태를 바꾸는 AI를 가리킨다. 핵심은 인지 → 계획(planning) → 제어(control)가 실제 물체·중력·마찰·센서 노이즈·불확실성 위에서 폐루프(closed loop)로 도는 것이다.

왜 지금 뜨는가? LLM/VLM이 웹 스케일로 언어·시각의 상식과 의미를 학습했고, 이 사전학습 지식을 로봇 행동으로 잇는 VLA(Vision-Language-Action) 접근이 2023~2025에 등장했다(RT-2 · OpenVLA · π0). 여기에 대규모 로봇 데이터 공개(Open X-Embodiment), 오픈 스택(LeRobot), 저가 하드웨어가 겹치며 "언어의 foundation model"을 "로봇의 foundation model"로 옮기려는 흐름이 만들어졌다.

기존 로봇 SW의 한계Physical AI(foundation-model) 접근
작업마다 규칙·모델을 따로 짬 → 물체·환경 하나 바뀌면 처음부터대규모·다양 데이터로 사전학습 후 적응 — 범용 정책(policy)
상태기계·수작업 인지·플래닝 파이프라인을 손으로 조립end-to-end 학습 정책 — 관측에서 행동까지 신경망이 사상
단일 로봇·단일 환경 전용 → 다른 로봇에 재사용 불가cross-embodiment 전이 — 여러 로봇 데이터가 서로를 밀어줌
시각·언어 상식 없음 → "행주로 닦아"의 의미를 모름VLM 상식 상속 — 물체 affordance·자연어 지시 이해
한 줄 정리

Physical AI = "웹에서 배운 VLM의 상식을, 로봇 데이터로 이어 붙여 물리 세계에서 인지·계획·제어를 폐루프로 수행하는 embodied AI."

01용어 사전 — 풀네임과 의미난이도 쉬움

이 분야는 로보틱스·제어·딥러닝 용어가 뒤섞인다. 아래 표는 용어 → 풀 네이밍 → 한 줄 의미로 정리했다. 나머지 섹션을 읽다 막히면 여기로 돌아오면 된다.

개념 축 — 인지·계획·제어

용어풀 네이밍개념 · 의미
Physical AI(= Embodied AI)물리 세계를 인지하고 몸체로 행동해 상태를 바꾸는 AI. 화면 밖에서 동작.
Perception인지센서 raw 데이터 → 유의미한 상태(물체 위치·자세·장면)로 추정하는 단계.
Planning계획목표와 현재 상태로부터 행동 시퀀스·궤적을 결정. 학습 정책이 여기 위치.
Control제어계획된 행동을 actuator 명령으로 변환. 보통 고빈도 폐루프(PID·MPC 등).
Closed-loop폐루프 (feedback control)행동 결과를 다시 센싱해 보정하는 순환. 오차·외란에 강함(open-loop 반대).
DoFDegrees of Freedom자유도. 로봇팔 말단은 보통 6-DoF(위치 3 + 자세 3) + 그리퍼 1.
Manipulation조작물체를 집고 옮기고 조립하는 과제군. Physical AI 벤치마크의 중심.
IMUInertial Measurement Unit관성측정장치. 가속도계·자이로로 자세·각속도·가속을 재는 센서. 센서퓨전의 자기운동 입력.
LiDARLight Detection and Ranging라이다. 레이저 비행시간으로 거리·3D 형상을 측정하는 깊이 센서.
Localization자기 위치 추정로봇이 환경 좌표계에서 자기 위치·자세를 추정하는 것. 인지 단계의 핵심.
Pose자세 (위치 + 방향)물체·말단의 3D 위치(3) + 방향(3)을 합친 6-DoF 표현. "6-DoF pose".
Sensor fusion센서 퓨전여러 센서를 시간·좌표 정합해 하나의 상태 추정(pose·위치)으로 통합하는 것.
PIDProportional-Integral-Derivative오차의 비례·적분·미분 항으로 제어 입력을 만드는 고전 저수준 폐루프 제어기.
MPCModel Predictive Control모델 예측 제어. 시스템 모델로 미래를 내다보고 매 스텝 최적 제어 입력을 푸는 제어기.

용어풀 네이밍개념 · 의미

Flow matching(플로 매칭)노이즈 → 연속 행동으로 정제하는 확산 계열 생성 기법. π0의 action head 방식.
Action head(행동 헤드)VLM backbone 위에 얹어 특징을 로봇 행동으로 변환·출력하는 부분.
Discrete token이산 토큰연속 행동을 구간으로 나눠 텍스트 토큰처럼 이산화한 행동 표현(RT-2·OpenVLA).
Covariate shift공변량 이동학습 시 관측 분포와 실행 시 관측 분포가 어긋나는 현상. imitation 실패의 근본 원인.
Compounding error누적 오차작은 오차가 다음 관측을 왜곡해 더 큰 오차를 부르는 악순환. covariate shift의 결과.
OODOut-of-Distribution학습 분포 밖의 입력·상황. 일반화·평가에서 반드시 확인하는 축.
Cross-embodiment(교차 임바디먼트)서로 다른 로봇 몸체의 데이터를 함께 학습해 전이하는 것. positive transfer의 전제.
Positive transfer(양의 전이)한 로봇·작업의 데이터가 다른 로봇·작업의 성능을 끌어올리는 효과.

02아키텍처 — Perception · Planning · Control 폐루프난이도 중간

Physical AI의 뼈대는 감지 → 인지 → 계획 → 제어 → 세계 → (피드백) → 감지로 도는 폐루프다. 고전 로보틱스는 이 세 단계를 분리된 모듈로 손수 조립했고, 최근 학습 정책은 여러 단계를 하나의 신경망으로 합친다.

  • Sensing(감지) — RGB 카메라·깊이(depth)·LiDAR·IMU·힘/토크 센서 + proprioception(관절 상태). raw 신호 계층.
  • Perception(인지) — raw → 상태 추정. 물체 6-DoF pose, 자기 위치(localization), 장면 이해. 센서 퓨전이 여기서 일어난다.
  • Planning(계획) — 목표 + 현재 상태 → 행동 시퀀스·궤적. 고전은 모션 플래너, 최근은 학습 정책 π / VLA가 담당.
  • Control(제어) — 계획을 actuator 명령으로. 안정성·안전을 위해 보통 수백 Hz~kHz 저수준 제어기가 따로 돈다.
면접 포인트 — 모듈러 vs end-to-end

고전 스택은 인지·플래너·제어기가 분리 모듈이라 디버깅·검증이 쉽지만 손수 조립·튜닝해야 한다. end-to-end 학습 정책/VLA는 관측에서 행동까지 하나의 망으로 합쳐 상식·일반화를 얻지만, 내부가 불투명하고 데이터가 많이 든다. 현실은 혼합이다 — 상위(인지~계획)는 학습, 하위(안전·저수준 제어)는 결정적 제어기.

03동작 원리 — 센서퓨전 · 학습 · VLA · sim2real난이도 중간

① 센서 퓨전과 인지

단일 센서로는 세계를 다 못 본다. 카메라는 색·질감, 깊이/LiDAR는 거리, IMU는 자세·가속, proprioception은 자기 관절 상태를 준다. 센서 퓨전은 이들을 시간·좌표 정합해 하나의 상태 추정(물체 pose, 자기 위치)으로 묶는다. 학습형 인지는 이 융합을 신경망 인코더가 담당하기도 한다.

② Imitation Learning vs RL

정책을 학습하는 두 큰 길이다. 실무는 둘을 섞는 경우가 많다.

축Imitation Learning (BC)Reinforcement Learning
학습 신호전문가 시연(teleoperation)의 관측→행동보상과 시행착오
장점안정·빠름 · 시연만 있으면 즉시 · 안전시연 없이 최적화 · 시연을 넘어서는 행동 발견 가능
약점covariate shift — 분포 밖에서 오차가 누적(compounding error)표본 비효율 · 보상 설계 난이도 · 실물 탐색 위험
주 무대실물 데이터(ACT · Diffusion Policy · 대부분의 VLA)주로 시뮬레이션, 실물은 미세조정에 제한적

대표 조합은 imitation으로 사전학습 → 소량 RL/실물 미세조정이다. VLA 계열도 기본은 대규모 시연을 imitation으로 학습한다(불확실: 최신 모델은 RL 요소를 일부 섞기도 하며 세부는 모델마다 다르다).

③ VLA — 멀티모달 입력에서 행동으로

VLA(Vision-Language-Action)는 웹에서 학습한 VLM backbone 위에 action head를 얹어, 이미지 + 자연어 지시(+ proprioception)를 받아 로봇 행동을 직접 출력한다. 행동을 표현하는 방식이 두 계열로 갈린다.

  • 이산 토큰(discrete token) — 행동을 텍스트 토큰처럼 이산화(예: 7-DoF를 각 256구간). VLM 어휘·구조를 그대로 재사용. RT-2(PaLI-X 55B)·OpenVLA(7B)가 이 계열. 큰 모델은 추론이 느릴 수 있다(RT-2-55B는 약 1~3Hz).
  • 연속 생성(flow matching / diffusion) — 노이즈에서 연속 행동 청크를 정제. π0는 PaliGemma VLM에 action expert를 붙여 flow matching으로 고빈도(최대 50Hz)·정교한(dexterous) 행동을 낸다.
  • action chunking — 한 스텝이 아니라 여러 미래 스텝을 한 번에 예측해 끊김·오차 누적을 줄인다(ACT에서 대중화).

④ Sim2Real gap

시뮬레이션은 값싸고 안전하며 데이터가 무한하지만, 물리·렌더링·센서 노이즈가 실물과 달라 시뮬에서 잘 되던 정책이 실물에서 실패한다. 이 격차가 reality gap이다. 완화 기법:

  • Domain randomization — 텍스처·조명·마찰·질량 등 시뮬 파라미터를 무작위화해, 실물을 "또 하나의 변형"으로 보게 만든다.
  • System identification — 실물 물리 파라미터를 측정해 시뮬을 실물에 맞춘다.
  • Digital twin — 실물 환경의 정밀 복제본에서 사전 학습·검증.
  • 실물 미세조정 — 시뮬 사전학습 정책을 소량의 실물 시연으로 보정.

05면접 핵심 정리 — 반드시 아는 것난이도 필수

면접 직전 이 섹션만 봐도 되도록, 반드시 그릴 아키텍처 · 파고드는 추가 설명 · 정의할 핵심 용어 · 아는지 확인하는 체크리스트를 한 곳에 모았다.

① 반드시 그리는 아키텍처

"Physical AI 시스템을 그려봐"라고 하면 이 그림이다. 세 축이 핵심 — 인지–정책(VLA)–제어 폐루프, 정책을 만드는 학습 방식(imitation vs RL), 그 정책을 떠받치는 대규모 데이터·sim2real.

② 추가 설명 — 면접관이 파고드는 지점

  • 학습이 폐루프의 어디에 들어가나 — 저수준 제어(모터 루프)는 결정적 제어기가 맡고, 학습 정책/VLA는 주로 인지~계획을 담당한다. end-to-end 정책은 인지~행동을 하나로 합치기도 한다.
  • imitation은 왜 실패하나 — 시연 분포를 벗어나면 오차가 다음 관측을 왜곡하고 다시 더 큰 오차를 부른다(covariate shift · compounding error). 그래서 데이터 다양성과 action chunking·소량 실물 보정이 중요하다.
  • VLA의 두 행동 표현 — 이산 토큰(RT-2·OpenVLA)은 VLM 구조를 그대로 재사용하나 큰 모델은 느릴 수 있고, flow matching/diffusion(π0)은 연속·고빈도·정교한 행동에 유리하다.
  • 왜 여러 로봇 데이터를 섞나 — Open X-Embodiment처럼 22개 로봇·1M+ 궤적을 모으면 한 로봇의 경험이 다른 로봇을 밀어주는 positive transfer가 생긴다(RT-X가 실증).
  • sim2real — reality gap의 원인(물리·렌더·노이즈 불일치)과 완화(domain randomization·system id·digital twin·실물 미세조정)를 한 문장으로 말할 수 있어야 한다.

06예상 질문 & 답변난이도 필수

Physical AI는 기존 LLM/VLM과 무엇이 다른가?
LLM/VLM은 텍스트·이미지라는 디지털 공간에서 입출력이 끝난다. Physical AI는 그 지식을 물리 세계의 행동으로 잇는다. 즉 카메라·센서로 인지하고, 정책이 행동을 계획하고, actuator로 제어해 세계의 상태를 실제로 바꾸며 그 결과를 피드백으로 다시 받는다(폐루프). VLA가 그 다리로, 웹에서 배운 VLM의 상식을 로봇 행동 출력에 연결한다.
Perception–Planning–Control 폐루프를 설명하고, 학습은 어디에 들어가나?
감지 → 인지(상태 추정) → 계획(행동 결정) → 제어(actuator 명령) → 세계 변화 → 새 관측으로 돌아오는 순환이다. 저수준 제어는 안정성·안전 때문에 수백 Hz~kHz의 결정적 제어기가 맡고, 학습 정책/VLA는 인지~계획을 담당한다. end-to-end 정책은 인지~행동을 하나의 신경망으로 합칠 수도 있지만, 현실은 상위=학습·하위=제어의 혼합이 흔하다.
Imitation Learning과 RL의 차이, 언제 무엇을 쓰나?
Imitation(BC)은 전문가 시연을 관측→행동으로 지도학습한다. 안정·빠르지만 시연 분포를 벗어나면 오차가 누적된다(covariate shift · compounding error). RL은 보상으로 시행착오해 시연 없이도, 때로는 시연을 넘어서는 행동을 찾지만 표본 비효율·보상 설계·실물 탐색 위험이 크다. 그래서 실물에선 imitation 사전학습 + 소량 RL/실물 미세조정이 흔하고, 순수 RL은 주로 시뮬에서 돌린다.
VLA 모델이 무엇이고, RT-2 · OpenVLA · π0는 어떻게 다른가?
VLA는 이미지 + 언어 지시를 받아 로봇 행동을 직접 출력하는 모델이다. RT-2(Google DeepMind, 2023)는 PaLI-X 55B VLM에서 행동을 이산 토큰으로 내며, 미학습 상황 성공률을 RT-1 대비 32%→62%로 올렸다(비공개 가중치). OpenVLA(2024)는 7B 오픈 모델로 Llama2 + DINOv2/SigLIP을 쓰고, Open X-Embodiment 970k 궤적으로 학습해 7배 작은 크기로 RT-2-X(55B)를 능가했다. π0(Physical Intelligence, 2024)는 PaliGemma에 action expert를 붙여 flow matching으로 최대 50Hz의 정교한 행동을 낸다.
π0가 이산 토큰 대신 flow matching을 쓰는 이유는?
이산 토큰은 행동을 구간으로 쪼개 표현력이 제한되고, 큰 VLM에서 토큰을 하나씩 자기회귀로 뽑으면 느려 고빈도 정교한 제어에 불리하다(RT-2-55B는 약 1~3Hz). π0는 diffusion 계열인 flow matching으로 노이즈에서 연속 행동 청크를 정제해, 접기·정리 같은 dexterous 작업을 최대 50Hz로 낸다. 한마디로 "연속·고빈도·정교함"을 위해 이산 토큰의 한계를 피한 설계다.
Sim2Real gap이 무엇이고 어떻게 줄이나?
시뮬의 물리·렌더링·센서 노이즈가 실물과 달라, 시뮬에서 되던 정책이 실물에서 실패하는 격차가 reality gap이다. 완화는 domain randomization(텍스처·조명·마찰·질량 무작위화로 실물을 또 하나의 변형처럼 보게 함), system identification(실물 파라미터로 시뮬 보정), digital twin(정밀 복제본에서 사전 검증), 그리고 소량 실물 미세조정이다.
여러 로봇의 데이터를 함께 쓰는 게 왜 중요한가(cross-embodiment)?
로봇마다 데이터를 새로 모으는 비용이 크고, 한 로봇·환경 전용 정책은 재사용이 안 된다. Open X-Embodiment는 22개 로봇 · 1M+ 궤적 · 527 skills를 하나의 표준 형식으로 모아, 한 로봇의 경험이 다른 로봇 성능을 끌어올리는 positive transfer를 보였다(RT-1-X · RT-2-X). 이는 웹 스케일 데이터가 언어·비전 foundation model을 가능케 한 것과 같은 논리를 로봇에 적용한 것이다.
저가 하드웨어로 Physical AI를 실습하려면? LeRobot은 무엇인가?
LeRobot(Hugging Face)은 PyTorch 기반 오픈 로보틱스 스택이다. 표준 데이터셋 형식(LeRobotDataset), 사전학습 정책(ACT · Diffusion Policy · TD-MPC 등 imitation/RL 계열과 π0 · SmolVLA 등 VLA), 그리고 저가 로봇 하드웨어(SO-100/SO-101) 지원을 묶어 데이터 수집 → 학습 → 실물 배포를 한 파이프라인으로 낮춘다. 개인·소규모 팀의 실습 진입 장벽을 크게 낮춘 스택이다.
