1.1 OVERVIEW
1.1 Claude Code 개요
터미널 네이티브 AI 코딩 에이전트
Claude Code는 Anthropic이 개발한 터미널 네이티브 AI 코딩 에이전트다. IDE 플러그인이나 브라우저 챗봇과 달리, 로컬 파일시스템·셸·Git에 직접 접근하여 실제 개발 워크플로우 안에서 동작한다.
Claude Code의 핵심 특징
  • 터미널(CLI)에서 직접 실행되며 로컬 파일을 읽고 쓴다
  • VS Code, IntelliJ 등 IDE의 확장 플러그인으로도 사용 가능
  • claude.ai/code 웹 인터페이스 및 모바일 앱과 연동
  • MCP(Model Context Protocol)로 외부 서비스와 통합
  • CLAUDE.md를 통해 프로젝트 컨텍스트를 세션 간 유지
📷
스크린샷 영역 Claude Code 터미널 첫 실행 화면 전체

1.1.1 macOS — 사전 요구사항
Node.js v22 권장 · Git v2.5x 이상
Claude Code를 설치하기 전에 Node.js와 Git 버전을 먼저 확인한다. Node.js가 없다면 nodejs.org에서 LTS 버전을 설치하고 npm이 함께 설치된다. Git이 없다면 아래 방법으로 설치한다.
node --version   # v22.x 권장 (v20 미만이면 업그레이드 필요)
git --version    # v2.5x 이상
# macOS — Xcode Command Line Tools (Git 포함)
xcode-select --install

# macOS — Homebrew로 설치
brew install git
# 최초 1회 사용자 정보 설정
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

1.3.2 settings.json — 팀 공유용 설정
Git 커밋해서 팀 전체에 공유하는 프로젝트 설정
{
  "model": "sonnet",
  "effortLevel": "high",
  "permissions": {
    "defaultMode": "acceptEdits",
    "allow": [
      "Bash(npm run *)",
      "Bash(git commit *)",
      "Bash(git status)",
      "Bash(git diff *)"
    ],
    "deny": [
      "Read(.env)",
      "Bash(git push *)",
      "Bash(rm -rf *)",
      "Bash(sudo *)"
    ]
  }
}
⚠️
deny가 항상 우선 deny 규칙이 allow 규칙보다 항상 우선 적용된다. ORM에서는 Bash(npx drizzle-kit drop *) 등을 deny에 추가.

1.3.4 ① default · ② acceptEdits — 일상 작업
안전 우선에서 시작 → acceptEdits로 흐름 가속
default — 파일 읽기만 자유. 편집·Bash·외부 호출은 매번 승인 요청. 흐름이 자주 끊기는 단점이 있어 시작점으로 사용 후 acceptEdits로 이동하는 흐름을 권장한다.
claude  # default로 시작
acceptEdits — 파일 읽기·편집 자동 승인. mkdir/touch/mv/cp 등 일반 파일시스템 명령도 작업 디렉토리 내에서 자동 승인. Bash 실행 명령은 여전히 매번 확인 요청. 상태 표시줄: ⏵⏵ accept edits on.
# Shift+Tab 한 번으로 진입
claude --permission-mode acceptEdits
🛡
안전 수칙 반드시 git checkpoint 후 사용. deny 룰 권장 — Bash(git push *), Bash(rm -rf *).
1.3.4 ① default · ② acceptEdits — 일상 작업 38

1.3.4 ③ plan · ④ auto — 계획 모드 & 자율 실행
Edit·Write 차단 vs 백그라운드 분류기 자동 승인
plan — 소스코드 편집(Edit·Write)은 차단된다. 단, 파일 읽기와 탐색용 Bash 명령 실행은 가능하다. Claude가 코드베이스를 분석하고 계획을 수립하지만 실제 소스코드를 변경하지 않는다. 상태 표시줄: ⏸ plan mode on.
# Shift+Tab 두 번으로 진입 (전체 세션 전환)
claude --permission-mode plan

# 단일 요청에만 plan 모드 적용
/plan 이 인증 모듈 리팩터링 계획 세워줘
auto — 모든 작업을 백그라운드 안전성 검사와 함께 자동 실행한다. 사람이 매번 승인하는 대신 백그라운드 분류기(classifier)가 요청 의도와 행동 일치 여부를 판단한다.
# 시작 플래그 필요
claude --enable-auto-mode
🔒
auto 제한 Pro 플랜 사용 불가. Max·Team·Enterprise·API 전용. Bedrock·Vertex·Foundry에서는 사용 불가.

1.3.4 ⑤ dontAsk · ⑥ bypassPermissions
사전 승인 화이트리스트 vs 모든 검사 생략
dontAsk — allow 리스트에 없는 모든 도구를 자동 거부한다. 거부 시 아무 메시지 없이 조용히 차단하므로 디버깅이 어렵다. Headless 자동화(claude -p), CI/CD 파이프라인 전용.
claude --permission-mode dontAsk
bypassPermissions — 권한 프롬프트를 건너뛰고 완전 자율 실행한다. 단, .git/.claude/.vscode/.idea/.husky 디렉토리 쓰기는 bypassPermissions에서도 여전히 확인을 요청한다 (리포 상태·설정 보호).
# bypassPermissions로 시작
claude --dangerously-skip-permissions

# 단일 명령 모드와 조합
claude -p "모든 테스트 실행하고 실패하면 수정해" --dangerously-skip-permissions
# --allowedTools 플래그 — bypassPermissions 없이 독립 사용
claude -p "린트 에러 수정해줘" \
  --allowedTools "Read,Write,Bash(npm run lint:*)"
⚠️
위험 경고 파일 삭제·프로젝트 범위 외 작업 위험. 격리 Docker·VM에서만 사용. CLAUDE.md + Hook으로 이중 제한 필요.
