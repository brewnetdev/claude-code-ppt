4.1.1 PROJECT — KMS
markflow — 지식관리 시스템으로 배우는 풀스택 개발
마크다운 기반 팀 KMS. SaaS 또는 온프레미스로 배포 가능한 오픈소스 — 폴더 단위 문서 연관 관계, 실시간 협업, RBAC를 핵심 기능으로 제공.
이 프로젝트로 습득하는 역량
  • AI와 협업해 요건 정의부터 배포까지 전 과정을 수행하는 능력
  • 모노레포 기반 풀스택 아키텍처 설계 및 구현 능력
  • TDD 기반의 안정적인 코드 작성 습관
  • SDD(Specification-Driven Development) 워크플로를 실무에 적용하는 능력
  • 명세 기반 AI 협업 개발에서 개발자의 오케스트레이터 역할 수립
📦
실무 복잡도  단순 TODO 앱보다 무겁다 — 워크스페이스 · 문서 버전 관리 · Closure Table · JWT · RBAC · Cloudflare R2 이미지 업로드 등 실제 SaaS 핵심 요소를 모두 포함.

4.3.1 데이터 설계 (1/2) 11
4.3.1 SCHEMA & CLOSURE TABLE
documents 스키마 + 카테고리 Closure Table
documents (Drizzle)
export const documents = pgTable('documents', {
  id: text('id').primaryKey(),
  title: text('title').notNull(),
  content: text('content').notNull().default(''),
  workspaceId: text('workspace_id')
    .notNull()
    .references(() => workspaces.id,
      { onDelete: 'cascade' }),
  categoryId: text('category_id')
    .references(() => categories.id,
      { onDelete: 'set null' }),
  authorId: text('author_id').notNull(),
  isPublished: boolean('is_published').default(false),
  createdAt: timestamp('created_at').defaultNow(),
});
category_closure (계층 패턴)
-- ancestor / descendant / depth
-- 자기 자신은 depth = 0
SELECT * FROM category_closure
WHERE ancestor = 'cat-001';
-- → cat-001 하위 모든 카테고리
--   한 번의 쿼리로 조회 가능
🌲
왜 Closure Table?  무한 깊이 지원, 단일 쿼리로 서브트리 조회. Adjacency List 대비 읽기 단순, 쓰기 시 약간의 오버헤드를 감수.
