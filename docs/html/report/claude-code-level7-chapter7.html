6.1.2 permissions.allow / deny — 평가 순서
Defense-in-Depth: PreToolUse Hook → deny → allow → ask → 권한 모드
권한 평가 순서: (1) PreToolUse Hook — 가장 먼저, 모든 룰을 무력화 가능 (2) deny 룰 — 매칭되면 즉시 차단 (3) allow 룰 — 매칭되면 즉시 승인 (4) ask 룰 — 사용자에게 prompt (5) 권한 모드 적용 — 위 단계에서 결정 안 났을 때
📘
핵심 주의점 Bash(curl:*) 단일 deny는 회피 가능 — wget·python urllib·nc로 우회. 진짜 네트워크 통제는 sandbox.network.allowedDomains가 책임

# .claude/hooks/check-injection.py
import json, re, sys

INJECTION_PATTERNS = [
    r"이전 지시를 무시",
    r"ignore previous instructions",
    r"~/\.ssh/",
    r"~/\.aws/",
    r"<!--.*?(curl|wget|exec).*?-->",
]

data = json.load(sys.stdin)
content = data.get("tool_input", {}).get("content", "")

for pattern in INJECTION_PATTERNS:
    if re.search(pattern, content, re.IGNORECASE | re.DOTALL):
        print(f"인젝션 패턴 감지: {pattern}", file=sys.stderr)
        sys.exit(2)   # 도구 실행 차단
sys.exit(0)

PreToolUse Hook — Write/Edit 전 비밀키 패턴 매칭하여 차단
# .claude/hooks/check-secrets.py
import json, re, sys

SECRET_PATTERNS = [
    (r"AKIA[0-9A-Z]{16}",                         "AWS Access Key"),
    (r"sk-ant-[A-Za-z0-9_-]{40,}",               "Anthropic API Key"),
    (r"sk-[A-Za-z0-9]{48}",                       "OpenAI API Key"),
    (r"-----BEGIN (RSA |EC )?PRIVATE KEY-----", "Private Key"),
    (r"ghp_[A-Za-z0-9]{36}",                      "GitHub Personal Token"),
    (r"postgres://[^:]+:[^@]+@",                  "Postgres Connection with Password"),
    (r"(?i)JWT_SECRET\s*=\s*['\"][^'\"]{8,}['\"]" ,"Hardcoded JWT Secret"),
]

data = json.load(sys.stdin)
content = data.get("tool_input", {}).get("content", "")

violations = [n for p, n in SECRET_PATTERNS if re.search(p, content)]
if violations:
    for v in violations:
        print(f"  - {v}", file=sys.stderr)
    sys.exit(2)
sys.exit(0)

6.3.1 /cost — 토큰 사용량 모니터링
/cost는 /usage의 alias. /stats도 동일. 공식 표준 명령은 /usage.
/cost (= /usage = /stats)는 세션 단위 토큰 사용량을 보여준다. Input tokens · Cached input (cache hit ratio) · Output tokens · 추정 비용을 표시. Pro/Max 플랜은 가격이 추정치 (실제 청구 영향 없음). API pay-as-you-go 플랜은 실제 발생 비용에 가까운 추정치.

  Output tokens:         18,732

같은 모듈 내 연속 작업 → /compact / 완전 다른 작업 → /clear
/compact: 요약 후 보존 — 같은 모듈 내에서 계속 작업할 때. /clear: 완전 초기화 — CLAUDE.md만 자동 로드. 결정 기준: 현재 작업과 다음 작업이 관련 있는가? YES → /compact (직전 결정사항 유지) / NO → /clear (이전 컨텍스트 없이 새 작업).
