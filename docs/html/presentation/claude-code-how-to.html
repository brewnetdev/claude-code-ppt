EPILOGUE · SDD · AI + 시너지
왜 AI와 SDD가 잘 맞는가 + TDD 결합
AI는 명확한 지시에 강하다 · 명세 자체가 AI를 위한 최고의 프롬프트.
❌ 모호한 지시
"TODO API 만들어줘"
AI 해석 분기:
REST? GraphQL? · 필드는? · 에러 응답 형식? · 인증?
매번 다른 결과
✅ 명세 기반 지시
"API_SPEC.md 읽고
POST /api/tickets 구현해줘"
요청 필드 · 응답 형식 · 에러 코드 모두 정의됨.
추측이 아닌 명세 기반 코드
📜 명세가 곧 프롬프트 — 하나의 문서, 두 역할
👤 개발자가 읽음
구현 방향 이해 · 리뷰 기준
🤖 AI가 읽음
코드 생성의 맥락 · 추측 방지
♻️ SDD + TDD 시너지
SDD — 무엇을 만들지 정의 (명세)
TDD — 만든 것이 동작하는지 검증 (테스트)
명세의 각 항목 → 테스트 케이스로 직접 변환
명세 → 테스트 → 구현 흐름 완성

EPILOGUE · SDD · Spec Kit 명령어
Spec Kit 8개 명령어 + 산출물
핵심 5개 + 선택 3개. 각 명령어가 다음 단계의 입력이 되는 명세 파일을 산출.
명령어 역할 사용 시점 산출물
/speckit.constitution 프로젝트 원칙 정의 프로젝트 최초 세팅 memory/constitution.md
/speckit.specify 요구사항 명세 작성 새 기능 개발 시작 전 [feature-name]/spec.md
/speckit.plan 기술 설계 수립 명세 확정 후 구현 전 plan.md · research.md · data-model.md · quickstart.md · contracts/
/speckit.tasks 구현 태스크 분해 설계 완료 후 코딩 직전 tasks.md
/speckit.implement 태스크 실행 태스크 분해 완료 후 실제 코드 파일
/speckit.clarify 선택 명세 모호함 제거 (AI 역질문) plan 전 명세 검증 spec.md에 Clarifications 섹션 추가
/speckit.analyze 선택 정합성·커버리지 분석 tasks 후 · implement 전 분석 리포트 (대화 출력)
/speckit.checklist 선택 요구사항 완전성·일관성 검증 품질 검증 시 검증 체크리스트 (대화 출력)
📦 설치: uv tool install specify-cli --from git+https://github.com/github/spec-kit.git
🚀 초기화: specify init my-project --ai claude

EPILOGUE · SDD · 수동 vs Spec Kit
언제 무엇을 쓸 것인가 + 실무 팁
기존 문서 표준 있다면 수동 워크플로 · 빠르게 시작하려면 Spec Kit.
✍️ 수동 워크플로
PRD · TRD · API_SPEC을 직접 작성
✓ 장점
  • 구조·포맷 완전 통제
  • 팀 기존 문서 표준 그대로
  • 도구 학습 불필요
✗ 단점
  • 문서 작성 시간 많음
  • 일관성 유지 어려움
  • AI 명세 전달이 수동
⚙️ Spec Kit 워크플로
슬래시 명령으로 AI와 함께 명세 생성·관리
✓ 장점
  • 명세 작성 빠름 (AI 초안)
  • 표준화된 구조 · 일관성
  • spec → plan → tasks → impl 자동화
  • 여러 AI 에이전트 공통
✗ 단점
  • 도구 설치·학습 필요
  • Spec Kit 템플릿 강제
  • 기존 문서 포맷과 호환 어려움

EPILOGUE · GSD · Spec Kit의 진화형
Get Shit Done — 무겁지만 강력한 SDD
Spec Kit과 같은 SDD 계열이지만 더 상세 · 더 무거움 — 복잡한 멀티-Phase 프로젝트에 유리.
🎯 핵심 문제: Context Rot — 컨텍스트 윈도우가 차면서 품질 저하. GSD는 매 단계를 새로운 200K 컨텍스트에서 실행해 이를 해결.
Spec Kit vs GSD 비교
축
📋 Spec Kit
🛠️ GSD (Get Shit Done)
명령어 수5 핵심 + 3 선택 = 8개50+ 명령어 (세분화)
토큰 소모가볍다 (~수K 시스템 프롬프트)무겁다 (~12K, --minimal로 700까지 절감 가능)
컨텍스트 전략단일 세션 흐름매 단계 200K 컨텍스트 격리 · 서브에이전트 분리
실행 모델순차 (constitution → specify → plan → tasks → implement)Wave 병렬 실행 · atomic git commit · 자동 검증
적합한 작업새 기능 · 단발 작업 · 빠른 시작복잡한 멀티-Phase 프로젝트 · 마일스톤 단위 · 장기 추적
학습 곡선낮음 — 8개만 외우면 됨높음 — 워크플로 + 설정 + 모델 프로파일 이해 필요
