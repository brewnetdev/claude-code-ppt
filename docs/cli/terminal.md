## 1. 터미널이란?

### 1.1 한 줄 설명

**마우스 없이 키보드로 컴퓨터에게 명령을 내리는 검은 창**입니다.

평소엔 Finder(macOS)나 탐색기(Windows)에서 폴더를 더블클릭해 열지만, 터미널에서는 글자로 입력합니다.

### 1.2 왜 명령어가 이렇게 짧고 이상한가요?

1970년대에 Unix가 만들어질 당시에는 **타자기처럼 한 글자씩 종이에 인쇄되는 단말기**를 썼습니다. 글자 하나 출력하는 데도 시간이 걸렸기 때문에 명령어를 최대한 짧게 줄였습니다.

그래서 `list`가 `ls`, `copy`가 `cp`, `remove`가 `rm`이 된 겁니다. 처음 보면 외계어 같지만, **영어 단어의 첫 글자나 줄임말이라는 걸 알면 훨씬 친근해집니다**.

### 1.3 터미널 여는 법

| OS | 방법 |
|----|------|
| macOS | `Command + Space` → "터미널" 입력 → Enter |
| Windows | `Win + R` → "wt" 또는 "powershell" 입력 |
| Linux | `Ctrl + Alt + T` |

### 1.4 처음 보는 화면 읽기

```
codevillain@MacBook-Pro ~ %
```

| 부분 | 의미 |
|------|------|
| `codevillain` | 사용자 이름 |
| `MacBook-Pro` | 컴퓨터 이름 |
| `~` | 현재 위치 (홈 폴더 = `/Users/사용자이름`) |
| `%` 또는 `$` | "여기에 명령어를 입력하세요" 신호 (프롬프트) |

> `%`나 `$`는 **타이핑하는 게 아닙니다**. 예제에서 `$ ls`라고 적혀 있으면 실제 입력은 `ls`만입니다.

## 2. 터미널 기본 명령어 (어원 포함)

### 2.1 위치·이동 관련

| 명령어 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `pwd` | **P**rint **W**orking **D**irectory | "현재 작업 중인 디렉토리 출력" | 내가 지금 어디 있는지 |
| `cd` | **C**hange **D**irectory | "디렉토리 변경" | 다른 폴더로 이동 |
| `ls` | **L**i**S**t | "목록" | 폴더 안 내용 보기 |

```bash
$ pwd
/Users/codevillain

$ ls
Desktop  Documents  Downloads  Pictures

$ cd Documents

$ pwd
/Users/codevillain/Documents
```

### 2.2 폴더 이동 상세 (cd)

| 명령어 | 의미 |
|--------|------|
| `cd 폴더이름` | 해당 폴더로 들어감 |
| `cd ..` | 상위 폴더로 (`..`는 "부모 폴더") |
| `cd ~` | 홈 폴더로 (`~`는 "내 홈") |
| `cd /` | 최상위(루트) 폴더로 (`/`는 시작점) |
| `cd -` | 직전에 있던 폴더로 (뒤로가기) |
| `cd` | 인자 없이도 홈 폴더로 |

> `.`은 "현재 폴더", `..`은 "한 단계 위 폴더"를 의미하는 약속된 기호입니다.
> `git add .`에서의 `.`도 같은 뜻 — "지금 이 폴더 전체".

### 2.3 파일·폴더 만들기

| 명령어 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `mkdir` | **M**a**K**e **DIR**ectory | "디렉토리 만들기" | 새 폴더 생성 |
| `touch` | (영단어 그대로) "건드리다" | 파일을 살짝 건드려 만들기 | 빈 파일 생성 |

```bash
$ mkdir my-project           # my-project 폴더 만들기
$ mkdir -p a/b/c             # 중간 폴더까지 한 번에 (parents)
$ touch README.md hello.txt  # 빈 파일 2개 만들기
```

### 2.4 복사·이동·삭제

| 명령어 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `cp` | **C**o**P**y | "복사" | 파일 복사 |
| `mv` | **M**o**V**e | "이동" | 파일 이동 또는 이름 변경 |
| `rm` | **R**e**M**ove | "제거" | 파일 삭제 |

```bash
$ cp README.md README-backup.md      # 복사
$ cp -r my-project backup-project    # 폴더 복사 (-r = recursive)
$ mv old.txt new.txt                 # 이름 변경
$ mv hello.txt Documents/            # 폴더로 이동
$ rm hello.txt                       # 파일 삭제
$ rm -r my-project                   # 폴더 통째로 삭제
```

> **⚠️ 절대 주의**
> 터미널의 `rm`은 **휴지통을 거치지 않습니다**. 한 번 지우면 끝입니다.
> 특히 `rm -rf /`는 "루트 폴더부터 강제로 다 지워라"라는 명령으로 컴퓨터를 통째로 날립니다. **절대 따라 치지 마세요**.

### 2.5 파일 내용 보기

| 명령어 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `cat` | con**CAT**enate | "이어 붙이기" | 파일 내용을 화면에 출력 |
| `less` | (영단어 그대로) `more`의 후속 — "less is more" 농담에서 유래 | 페이지 단위로 보기 |
| `head` | "머리" | 파일 앞부분 |
| `tail` | "꼬리" | 파일 뒷부분 |

```bash
$ cat README.md           # 전체 출력
$ less long-file.txt      # 페이지 넘기며 보기 (q로 종료)
$ head -5 file.txt        # 앞 5줄
$ tail -5 file.txt        # 뒤 5줄
$ tail -f server.log      # 실시간 로그 (-f = follow)
```

> **재미있는 어원**: `cat`은 고양이가 아니라 **concatenate(연결)**의 약어입니다.
> 원래 여러 파일을 이어 붙이는 용도였는데, 한 파일만 출력해도 작동해서 "파일 내용 보기"로 더 많이 쓰입니다.
>
> **`less`의 농담**: 옛날엔 `more` 명령어가 있었는데(한 페이지씩 보고 더 보려면 스페이스), 그 후속으로 만든 게 `less`입니다. "less is more"라는 영어 격언을 패러디한 이름이에요.

### 2.6 도움말·검색

| 명령어 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `man` | **MAN**ual | "설명서" | 명령어의 전체 매뉴얼 |
| `which` | (영단어 그대로) "어느 것?" | 명령어가 설치된 경로 |
| `grep` | **G**lobally search **R**egular **E**xpression and **P**rint | 패턴 검색 |
| `find` | (영단어 그대로) "찾다" | 파일 찾기 |

```bash
$ man ls                          # ls의 전체 매뉴얼 (q로 종료)
$ which python                    # python이 설치된 위치
$ grep "error" server.log         # server.log에서 "error" 찾기
$ find . -name "*.md"             # 현재 폴더 아래 .md 파일 모두
```

> **`grep` 어원이 재미있음**: 원래 ed 편집기의 명령 `g/re/p`(global / regular expression / print = 전체에서 정규표현식으로 찾아 출력)에서 따왔습니다. 이게 너무 유용해서 별도 명령어로 독립한 것.

### 2.7 기타 자주 쓰는 것

| 명령어 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `echo` | (영단어 그대로) "메아리" | 글자 출력 |
| `clear` | (영단어 그대로) "비우다" | 화면 지우기 |
| `history` | (영단어 그대로) "기록" | 입력한 명령어 기록 |
| `exit` | (영단어 그대로) "나가다" | 터미널 종료 |
| `sudo` | **S**uper**U**ser **DO** | "최고 권한자로 실행" |

```bash
$ echo "Hello"                    # Hello
$ clear                           # 화면 깨끗이
$ history                         # 지난 명령어 목록
$ sudo apt install gh             # 관리자 권한으로 설치
$ exit                            # 터미널 닫기
```

> **`sudo` 강조**: 비밀번호를 물어보는 명령은 거의 다 `sudo`입니다.
> **시스템에 영향을 주는 명령**이라는 뜻이니 신중하게 쓰세요.

## 3. 옵션(플래그) 약어 정리

명령어 뒤에 붙는 `-r`, `-f`, `-a` 같은 게 **옵션(플래그)**입니다. 이것도 다 영어 단어의 약어입니다.

### 3.1 가장 자주 만나는 옵션

| 옵션 | 약어 | 풀이 | 의미 |
|------|------|------|------|
| `-r` / `-R` | **R**ecursive | "재귀적으로" | 폴더 안쪽까지 |
| `-f` | **F**orce | "강제로" | 확인 없이 실행 |
| `-a` | **A**ll | "모두" | 숨김 파일 포함 모두 |
| `-l` | **L**ong | "긴 형식" | 자세히 표시 |
| `-v` | **V**erbose | "수다스럽게" | 진행 상황 출력 |
| `-h` | **H**uman-readable / **H**elp | "사람이 읽기 좋게" / "도움말" | 1024 대신 1K / 도움말 |
| `-i` | **I**nteractive | "물어보면서" | 작업 전에 확인 |
| `-p` | **P**arents / **P**reserve | "부모까지" / "유지" | 중간 폴더까지 만들기 |

### 3.2 합쳐 쓰기

옵션은 합쳐 쓸 수 있습니다.

```bash
$ ls -l -a       # 같음
$ ls -la         # 이렇게도 됨
$ ls -al         # 순서 상관없음
```

### 3.3 `rm -rf`의 의미를 분해해보면

```
rm   -r   -f
 │    │    │
 │    │    └─ Force      : 묻지 말고
 │    └────── Recursive  : 폴더 안쪽까지
 └─────────── Remove     : 삭제하라

→ "묻지 말고 폴더 안쪽 전부 지워라"
```

이게 왜 위험한지 보이시죠? 묻지도 않고, 폴더 통째로, 휴지통도 거치지 않습니다.

### 3.4 다른 예시들

```bash
$ ls -la         # List   : Long + All        → 자세히 + 숨김파일까지
$ cp -rv a b     # Copy   : Recursive + Verbose → 폴더 통째 복사 + 진행상황 출력
$ mkdir -p a/b/c # Make Directory : Parents  → 중간 폴더까지 만들기
$ rm -i file     # Remove : Interactive      → 지울 때 물어봐줘
```

`-i` 옵션은 비개발자에게 안전장치로 추천할 만합니다. 다음을 `~/.zshrc`에 추가하면 `rm` 사용 시 항상 물어봅니다:

```bash
alias rm='rm -i'
```

## 4. 터미널 키보드 단축키 (macOS)

> 마우스로 글자 위치를 바꾸는 대신 키보드만으로 빠르게 움직이는 방법입니다.
> Windows/Linux 사용자는 `Command` → `Ctrl`로 대부분 동일하게 작동합니다.

### 4.1 단축키 약어 (먼저 알아두면 외우기 쉬움)

| 키 | 약어 | 의미 |
|----|------|------|
| `Ctrl + A` | **A**head / 줄의 시작 | 줄 맨 앞 |
| `Ctrl + E` | **E**nd | 줄 맨 끝 |
| `Ctrl + U` | **U**nix-cut to start | 줄 전체 삭제 |
| `Ctrl + K` | **K**ill to end | 끝까지 삭제 |
| `Ctrl + W` | **W**ord delete | 단어 삭제 |
| `Ctrl + C` | **C**ancel | 취소 / 강제 종료 |
| `Ctrl + D` | **D**one (EOF) | 입력 종료 |
| `Ctrl + L` | c**L**ear | 화면 지우기 |
| `Ctrl + R` | **R**everse search | 히스토리 검색 |
| `Cmd + F` | **F**ind | 찾기 |
| `Cmd + T` | **T**ab | 새 탭 |
| `Cmd + N` | **N**ew | 새 창 |

### 4.2 커서(삽입점) 이동

| 동작 | 단축키 |
|------|-------|
| 삽입점으로 위치 변경 | `Option` 키를 누른 상태로 포인터를 새 위치로 이동 |
| 명령어 라인의 **시작**으로 | `Control + A` (**A**head) |
| 명령어 라인의 **끝**으로 | `Control + E` (**E**nd) |
| 한 **문자** 앞으로 | `→` (오른쪽 화살표) |
| 한 **문자** 뒤로 | `←` (왼쪽 화살표) |
| 한 **단어** 앞으로 | `Option + →` |
| 한 **단어** 뒤로 | `Option + ←` |
| 단어의 **끝**으로 | `Option + D` (Option을 Meta 키로 설정 시) |
| 단어의 **시작**으로 | `Control + W` (**W**ord) |

### 4.3 삭제

| 동작 | 단축키 |
|------|-------|
| 라인 전체 삭제 | `Control + U` |
| 커서부터 라인 끝까지 삭제 | `Control + K` (**K**ill) |
| 커서 앞 한 단어 삭제 | `Control + W` (**W**ord) |
| 한 글자 삭제 (뒤로) | `Delete` |

### 4.4 검색

| 동작 | 단축키 |
|------|-------|
| 찾기 | `Command + F` (**F**ind) |
| 다음 찾기 | `Command + G` |
| 이전 찾기 | `Command + Shift + G` |
| 이전 명령어 검색 (히스토리) | `Control + R` (**R**everse search) |

### 4.5 화면·창

| 동작 | 단축키 |
|------|-------|
| 화면 지우기 | `Command + K` 또는 `Control + L` (c**L**ear) |
| 새 탭 열기 | `Command + T` (**T**ab) |
| 새 창 열기 | `Command + N` (**N**ew) |
| 탭 간 이동 | `Command + Shift + ←/→` |
| 글자 크게 | `Command + +` |
| 글자 작게 | `Command + -` |

### 4.6 실행 제어 (필수)

| 동작 | 단축키 |
|------|-------|
| 실행 중인 명령 강제 종료 | `Control + C` (**C**ancel) |
| 입력 종료 (EOF) | `Control + D` (**D**one) |
| 명령 일시정지 | `Control + Z` |
| 이전 명령어 불러오기 | `↑` (위 화살표) |
| 다음 명령어 | `↓` (아래 화살표) |

> **강사 강조 포인트**
> 비개발자가 가장 많이 막히는 상황은 **명령이 안 끝나서 프롬프트가 안 돌아올 때**입니다.
> 일단 `Control + C`(**C**ancel)를 누르라고 알려주면 90%는 해결됩니다.

## 5. Git 기본 명령어 (어원 포함)

### 5.1 Git이 뭔가요?

**파일의 변경 이력을 저장하고, 협업할 수 있게 해주는 도구**입니다.

> **이름 어원**: 만든 사람 리누스 토르발스(리눅스 창시자)가 자신을 "git"이라고 농담했는데, 영국 슬랭으로 "고집 센 영감"이라는 뜻입니다. 진지한 의미보다 농담조로 붙인 이름입니다.

### 5.2 Git 용어 약어

| 용어 | 풀이 | 의미 |
|------|------|------|
| **repo** | **REPO**sitory | "저장소" — 프로젝트 폴더 |
| **commit** | (영단어) "약속하다, 맡기다" | "이 상태를 영구히 기록" |
| **branch** | (영단어) "가지" | "평행 작업 라인" |
| **merge** | (영단어) "합치다" | "두 가지를 하나로" |
| **HEAD** | (영단어) "머리" | "지금 보고 있는 위치" |
| **origin** | (영단어) "원점" | "원격 저장소의 기본 별명" |
| **main** / **master** | "메인" / "주" | "기본 브랜치 이름" |

### 5.3 처음 한 번만 하는 설정

```bash
git config --global user.name "Code Villain"
git config --global user.email "codevillain@example.com"
```

> `config`는 **CONFIG**uration(설정)의 약어. `--global`은 "이 컴퓨터 전체에 적용".

### 5.4 핵심 흐름: 작업 → 저장 → 공유

```
   [작업 폴더]                                  [GitHub]
   파일 수정
     │
     │ git add        (저장할 파일 고르기)
     ▼
   [Staging Area]    ← "무대"라는 뜻, 발표하기 전 대기 장소
     │
     │ git commit -m "메시지"     (세이브 포인트 만들기)
     ▼
   [로컬 저장소]
     │
     │ git push       (GitHub에 올리기)
     ▼
   [원격 저장소]
```

### 5.5 저장소 만들기·가져오기

| 명령어 | 약어/어원 | 의미 |
|--------|----------|------|
| `git init` | **INIT**ialize "초기화" | 이 폴더를 Git 저장소로 |
| `git clone` | (영단어) "복제" | 원격 저장소를 통째로 받기 |

```bash
$ git init                                              # 새 저장소 만들기
$ git clone https://github.com/villainscode/my-repo.git  # 원격 저장소 복제
```

### 5.6 상태 확인

| 명령어 | 약어/어원 | 의미 |
|--------|----------|------|
| `git status` | (영단어) "상태" | 지금 어떤 파일이 바뀌었나? |
| `git log` | (영단어) "기록" | 커밋 기록 보기 |
| `git diff` | **DIFF**erence "차이" | 무엇이 어떻게 바뀌었나? |

```bash
$ git status
On branch main
Changes not staged for commit:
  modified:   README.md

$ git log --oneline    # 한 줄씩 간단히
4f1d2a3 README 첫 작성
a2b3c4d 초기 설정

$ git diff             # 줄 단위로 변경사항
```

> **막혔을 때 일단 `git status`**. Git이 친절하게 "이런 상태이고, 이렇게 하면 됩니다"라고 알려줍니다.

### 5.7 저장하기 (add → commit)

| 명령어 | 약어/어원 | 의미 |
|--------|----------|------|
| `git add` | (영단어) "추가" | 저장 후보 목록에 올리기 |
| `git commit` | (영단어) "약속하다" | 세이브 포인트 만들기 |
| `-m` | **M**essage | 커밋 메시지 |
| `-a` | **A**ll | 모든 변경 자동 add |

```bash
$ git add README.md              # 하나만
$ git add .                      # 전부 (`.`은 "현재 폴더 전체")
$ git commit -m "README 작성"    # 커밋 + 메시지
$ git commit -am "버그 수정"     # add + commit 한 번에 (기존 파일만)
```

**좋은 커밋 메시지:**
- ✅ `로그인 버튼 색상 변경`
- ✅ `회원가입 이메일 검증 추가`
- ❌ `수정함`
- ❌ `ㅇㅇ`

### 5.8 공유하기 (push / pull)

| 명령어 | 어원 | 의미 |
|--------|------|------|
| `git push` | (영단어) "밀다" | 내 변경을 원격으로 밀어 올리기 |
| `git pull` | (영단어) "당기다" | 원격 변경을 내 쪽으로 당겨오기 |
| `git fetch` | (영단어) "가져오다" | 원격 변경만 확인 (병합 안 함) |
| `git remote` | (영단어) "원격" | 원격 저장소 설정 |

```bash
# 처음 한 번
$ git remote add origin https://github.com/villainscode/my-blog.git
$ git push -u origin main      # -u = Upstream "기본 연결 설정"

# 이후엔
$ git push
$ git pull
```

> **`origin`이란?**: 원격 저장소의 기본 별명입니다. "출신지/원점"이라는 뜻에서 옴.
> 다른 이름을 줘도 되지만 관례적으로 `origin`을 씁니다.

### 5.9 브랜치 (가지)

브랜치(branch) = "가지". 평행 우주에서 작업하기.

| 명령어 | 어원 | 의미 |
|--------|------|------|
| `git branch` | "가지" | 브랜치 목록 |
| `git switch` | "전환" | 다른 브랜치로 이동 |
| `git switch -c` | **C**reate | 만들고 바로 이동 |
| `git merge` | "합치다" | 다른 브랜치를 현재로 합치기 |

```bash
$ git branch                       # 목록
$ git switch -c feature/login      # 새 브랜치 만들고 이동
# ... 작업 후 commit ...
$ git switch main                  # 본 줄기로 복귀
$ git merge feature/login          # 작업 결과 합치기
```

### 5.10 실수 복구

| 명령어 | 어원 | 의미 |
|--------|------|------|
| `git restore` | "복원" | 변경 되돌리기 |
| `git restore --staged` | "무대에서 내리기" | `git add` 취소 |
| `git commit --amend` | "수정" | 직전 커밋 수정 |

```bash
$ git restore README.md              # 변경한 내용 버리기
$ git restore --staged README.md     # add를 취소
$ git commit --amend -m "새 메시지"  # 직전 커밋 메시지 수정
```

> **⚠️ 주의**: `git reset --hard`, `git push --force`는 데이터를 영구히 날릴 수 있습니다.
> 비개발자 단계에서는 피하시고, 막히면 `git status` 안내를 따르세요.

### 5.11 자주 만나는 시나리오

**시나리오 1: 처음 GitHub에 올리기**

```bash
cd my-project
git init                                                  # 저장소 초기화
git add .                                                 # 전부 추가
git commit -m "첫 커밋"                                   # 첫 커밋
git remote add origin https://github.com/USER/REPO.git    # 원격 연결
git branch -M main                                        # 기본 브랜치 이름을 main으로
git push -u origin main                                   # 업스트림 설정하며 푸시
```

**시나리오 2: 매일의 작업 루틴**

```bash
git pull                # 최신 받기
# ... 작업 ...
git status              # 상태 확인
git add .
git commit -m "오늘 작업"

```

## 6. nano 편집기 사용법

### 6.1 nano가 뭔가요?

**터미널에서 쓰는 가장 쉬운 편집기**입니다. 메모장과 비슷해서 비개발자에게 가장 친절합니다.

> **이름 어원**: 원래 `pico` 편집기의 후계자로 만들어졌고, "pico보다 더 작다"는 의미로 "nano"(나노 = 10⁻⁹, pico = 10⁻¹²보다 작은 단위)라고 이름 붙였습니다.
> 사실 기능은 pico보다 많은데 농담조 이름입니다.

화면 아래에 단축키가 항상 표시되니까 외울 필요도 없습니다.

### 6.2 화면 보는 법

nano를 열면 화면 아래에 이런 게 보입니다:

```
^G Help    ^O Write Out    ^W Where Is    ^K Cut
^X Exit    ^R Read File    ^\ Replace     ^U Paste
```

`^`는 `Ctrl` 키를 의미합니다. 즉 `^X`는 `Ctrl + X`.

### 6.3 열기·저장·종료 (이것만 알면 됨)

| 단축키 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `nano 파일명` | | | 파일 열기 (없으면 새로 만듦) |
| `Ctrl + O` | **O**ut | "Write Out" — 써서 내보내기 | 저장 |
| `Ctrl + X` | e**X**it | "나가다" | 종료 |
| `Ctrl + G` | **G**et help | "도움 받기" | 도움말 |

**가장 흔한 흐름:**

```bash
$ nano hello.txt
# 글자 입력...
# Ctrl + O → Enter (저장)
# Ctrl + X (종료)
```

### 6.4 편집 기능

| 단축키 | 약어 | 풀이 | 의미 |
|--------|------|------|------|
| `Ctrl + K` | **K**ut(=Cut) | "잘라내기" | 한 줄 잘라내기 |
| `Ctrl + U` | **U**ncut | "되돌려 붙이기" | 붙여넣기 |
| `Ctrl + W` | **W**here is | "어디 있나?" | 찾기 |
| `Ctrl + \` | | | 찾아 바꾸기 |
| `Ctrl + _` | | | 특정 줄 번호로 이동 |
| `Ctrl + A` | **A**head | "맨 앞" | 줄 시작으로 |
| `Ctrl + E` | **E**nd | "끝" | 줄 끝으로 |

### 6.5 비개발자가 가장 자주 만나는 상황

**저장하고 나갈 때 묻는 메시지:**

```
Save modified buffer?
 Y Yes    N No    ^C Cancel
```

→ `Y` 입력 후 Enter

**파일 이름 확인:**

```
File Name to Write: hello.txt
```

→ 그대로 Enter 누르면 그 이름으로 저장

## 7. vi 편집기 사용법

### 7.1 vi가 뭔가요?

**거의 모든 Unix 계열 시스템에 기본 설치되어 있는 편집기**입니다.

> **이름 어원**:
> - `vi`는 **VI**sual editor의 약어
> - 그 이전엔 `ed`(editor), `ex`(extended editor)가 있었고, vi는 "ex의 비주얼 모드"로 시작
> - 요즘은 대부분 `vim`이 설치되어 있는데, **VI iM**proved (vi 개선판)의 약어
> - macOS에서 `vi`를 치면 사실 `vim`이 실행됩니다

### 7.2 ⚠️ 가장 먼저 외워야 할 것: 탈출법

vi에 갇혔다면:

```
Esc 키 → : 입력 → q! 입력 → Enter
```

읽기: "Escape, 콜론, q, 느낌표, 엔터" = **저장 안 하고 강제 종료**

| 입력 | 약어 | 의미 |
|------|------|------|
| `Esc` | **ESC**ape | "탈출, 빠져나오기" → 일반 모드로 |
| `:` | | "이제 명령을 칠 거예요" |
| `q` | **Q**uit | "끝내기" |
| `!` | | "강제로! 무조건!" |

이것만 알아도 패닉에서 벗어날 수 있습니다.

### 7.3 vi의 핵심 개념: 모드(Mode)

vi는 다른 편집기와 다릅니다. **모드**라는 게 있어요.

```
        [일반 모드 (Normal)]
        - 화살표로 이동
        - 명령어 입력
              │
        ┌─────┴─────┐
        │           │
   i 누르면     : 누르면
        │           │
        ▼           ▼
  [입력 모드]   [명령 모드]
   글자 입력     :w 저장
   가능         :q 종료
        │           │
   Esc 누르면  Enter 후 자동 복귀
        │           │
        └─────┬─────┘
              ▼
        [일반 모드]
```

**비유**: 일반 모드는 "리모컨", 입력 모드는 "글쓰기 펜".
글을 쓰려면 펜으로 바꾸고(`i`), 다 쓰면 다시 리모컨으로 돌아갑니다(`Esc`).

### 7.4 vi 사용 흐름 (절대 외울 것)

```
1. vi 파일명           ← 파일 열기
2. i                   ← 입력 모드로 (글자 쓸 수 있음)
3. (글자 입력)
4. Esc                 ← 일반 모드로 복귀
5. :wq                 ← 저장하고 종료
```

이게 vi의 80%입니다.

### 7.5 모드 전환 (어원과 함께)

| 키 | 약어 | 풀이 | 동작 |
|----|------|------|------|
| `i` | **I**nsert | "삽입" | 커서 **앞**에서 입력 시작 |
| `a` | **A**ppend | "덧붙이기" | 커서 **뒤**에서 입력 시작 |
| `o` | **O**pen line | "새 줄 열기" | **아래** 새 줄 만들어 입력 |
| `Esc` | **ESC**ape | "탈출" | 일반 모드로 복귀 |

### 7.6 저장·종료 (`:` 명령)

`Esc`로 일반 모드 들어간 후:

| 명령 | 약어 | 의미 |
|------|------|------|
| `:w` | **W**rite | 저장 |
| `:q` | **Q**uit | 종료 |
| `:wq` | **W**rite + **Q**uit | 저장하고 종료 |
| `:q!` | **Q**uit + 강제 | 저장 안 하고 강제 종료 |
| `:w 파일명` | | 다른 이름으로 저장 |
| `ZZ` | | `:wq`와 같음 (shift+z 두 번) |

### 7.7 이동 (일반 모드에서)

| 키 | 어원 | 의미 |
|----|------|------|
| `h` `j` `k` `l` | 키보드 위치상 좌·하·상·우 | 화살표 대용 (옛날엔 화살표 키가 없었음) |
| `0` | (숫자 0) | 줄의 맨 앞 |
| `$` | (정규식 약속) | 줄의 맨 끝 |
| `gg` | **G**o to top | 파일 맨 위 |
| `G` | **G**o to end | 파일 맨 아래 |
| `숫자G` | | 그 줄 번호로 이동 (`10G` = 10번째 줄) |
| `Ctrl + f` | **F**orward | 한 페이지 아래 |
| `Ctrl + b` | **B**ackward | 한 페이지 위 |

> 비개발자에게 `hjkl` 대신 그냥 **화살표 키**를 쓰라고 하세요. 같은 결과입니다.

### 7.8 편집 (일반 모드에서)

| 키 | 어원 | 의미 |
|----|------|------|
| `x` | (영단어) "지우다" | 커서 위치 한 글자 삭제 |
| `dd` | **D**elete + **D**elete | 한 줄 삭제 (잘라내기) |
| `yy` | **Y**ank + **Y**ank | 한 줄 복사 (yank = 잡아당기다) |
| `p` | **P**aste / **P**ut | 붙여넣기 |
| `u` | **U**ndo | 실행 취소 |
| `Ctrl + r` | **R**edo | 다시 실행 |

### 7.9 찾기 (일반 모드에서)

| 키 | 어원 | 의미 |
|----|------|------|
| `/검색어` | | 아래 방향으로 찾기 |
| `?검색어` | | 위 방향으로 찾기 |
| `n` | **N**ext | 다음 결과 |
| `N` | (Shift+n) | 이전 결과 |
| `:%s/이전/이후/g` | **s**ubstitute / **g**lobal | 전체에서 바꾸기 |

### 7.10 비개발자가 가장 자주 만나는 시나리오

**시나리오 1: `git commit` 했더니 vi가 떴어요!**

```
git commit
→ vi 화면 등장
→ i 누르고 메시지 입력
→ Esc
→ :wq 입력 후 Enter
→ 끝
```

**시나리오 2: 망쳤어요, 처음부터 다시!**

```
Esc → :q! → Enter
```

저장 안 하고 나갑니다. 그리고 다시 `vi 파일명`으로 열면 됩니다.

### 7.11 nano vs vi 비교

| 항목 | nano | vi |
|------|------|----|
| 배우기 쉬움 | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| 어디에나 설치 | 보통 | 거의 항상 |
| 단축키 표시 | 화면 아래 항상 | 외워야 함 |
| 비개발자 추천 | 첫 편집기로 추천 | 탈출법만 알면 됨 |

> **결론**: 평소엔 **nano**를 쓰고, 갑자기 **vi**가 뜨면 `:wq`나 `:q!`로 빠져나오세요.

### 8.6 막혔을 때 체크리스트

1. 일단 `Control + C` (Cancel) 눌러봤나요?
2. `git status`로 Git이 뭐라고 하는지 봤나요?
3. vi에 갇혔으면 `Esc → :q! → Enter`
4. 그래도 안 되면 터미널 창을 닫고 새로 여세요
5. 정 안 되면 오류 메시지를 그대로 복사해서 검색하세요
