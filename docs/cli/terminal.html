비개발자를 위한 입문 가이드 — 명령어 어원으로 자연스럽게 외우기

1. 터미널이란?

1.1 한 줄 설명

마우스 없이 키보드로 컴퓨터에게 명령을 내리는 검은 창입니다. 평소엔 Finder(macOS)나 탐색기(Windows)에서 폴더를 더블클릭해 열지만, 터미널에서는 글자로 입력합니다.

1.2 왜 명령어가 이렇게 짧고 이상한가요?

1970년대 Unix 시절 타자기처럼 한 글자씩 종이에 인쇄되는 단말기를 썼습니다. 글자 하나 출력하는 데도 시간이 걸렸기 때문에 명령어를 최대한 짧게 줄였습니다.

그래서 list가 ls, copy가 cp, remove가 rm이 된 겁니다. 영어 단어의 첫 글자나 줄임말이라는 걸 알면 훨씬 친근해집니다.

1.3 터미널 여는 법

OS방법
macOSCommand + Space → "터미널" 입력 → Enter
WindowsWin + R → "wt" 또는 "powershell" 입력
LinuxCtrl + Alt + T

1.4 처음 보는 화면 읽기

프롬프트 예시
codevillain@MacBook-Pro ~ %
부분의미
codevillain사용자 이름
MacBook-Pro컴퓨터 이름
~현재 위치 (홈 폴더 = /Users/사용자이름)
% 또는 $"여기에 명령어를 입력하세요" 신호 (프롬프트)
💡
중요%나 $는 타이핑하는 게 아닙니다. 예제에서 $ ls라고 적혀 있으면 실제 입력은 ls만입니다.

2. 터미널 기본 명령어 (어원 포함)

2.1 위치·이동 관련

명령어약어풀이의미
pwdPrint Working Directory현재 작업 중인 디렉토리 출력내가 지금 어디 있는지
cdChange Directory디렉토리 변경다른 폴더로 이동
lsLiSt목록폴더 안 내용 보기
 
Terminal
$ pwd
/Users/codevillain

$ ls
Desktop  Documents  Downloads  Pictures

$ cd Documents

$ pwd
/Users/codevillain/Documents

2.2 폴더 이동 상세 (cd)

명령어의미
cd 폴더이름해당 폴더로 들어감
cd ..상위 폴더로 (..는 "부모 폴더")
cd ~홈 폴더로 (~는 "내 홈")
cd /최상위(루트) 폴더로 (/는 시작점)
cd -직전에 있던 폴더로 (뒤로가기)
cd인자 없이도 홈 폴더로
.은 "현재 폴더", ..은 "한 단계 위 폴더"를 의미하는 약속된 기호입니다. git add .에서의 .도 같은 뜻 — "지금 이 폴더 전체".

2.3 파일·폴더 만들기

명령어약어풀이의미
mkdirMaKe DIRectory디렉토리 만들기새 폴더 생성
touch(영단어) "건드리다"파일을 살짝 건드려 만들기빈 파일 생성
 
Terminal
$ mkdir my-project           # my-project 폴더 만들기
$ mkdir -p a/b/c             # 중간 폴더까지 한 번에 (parents)
$ touch README.md hello.txt  # 빈 파일 2개 만들기

2.4 복사·이동·삭제

명령어약어풀이의미
cpCoPy복사파일 복사
mvMoVe이동파일 이동 또는 이름 변경
rmReMove제거파일 삭제
 
Terminal
$ cp README.md README-backup.md      # 복사
$ cp -r my-project backup-project    # 폴더 복사 (-r = recursive)
$ mv old.txt new.txt                 # 이름 변경
$ mv hello.txt Documents/            # 폴더로 이동
$ rm hello.txt                       # 파일 삭제
$ rm -r my-project                   # 폴더 통째로 삭제
⚠️
절대 주의터미널의 rm은 휴지통을 거치지 않습니다. 한 번 지우면 끝입니다. 특히 rm -rf /는 "루트 폴더부터 강제로 다 지워라"라는 명령으로 컴퓨터를 통째로 날립니다. 절대 따라 치지 마세요.

2.5 파일 내용 보기

명령어약어풀이의미
catconCATenate이어 붙이기파일 내용을 화면에 출력
less(영단어) "less is more" 농담more의 후속페이지 단위로 보기
head"머리"파일 앞부분앞 N줄
tail"꼬리"파일 뒷부분뒤 N줄
 
Terminal
$ cat README.md           # 전체 출력
$ less long-file.txt      # 페이지 넘기며 보기 (q로 종료)
$ head -5 file.txt        # 앞 5줄
$ tail -5 file.txt        # 뒤 5줄
$ tail -f server.log      # 실시간 로그 (-f = follow)
재미있는 어원 cat은 고양이가 아니라 concatenate(연결)의 약어입니다. less의 농담 — 옛날엔 more 명령어가 있었는데, 그 후속으로 만든 게 less입니다. "less is more"라는 영어 격언을 패러디한 이름이에요.

2.6 도움말·검색

명령어약어풀이의미
manMANual설명서명령어의 전체 매뉴얼
which(영단어) "어느 것?"설치 경로명령어가 설치된 위치
grepGlobally search RegExp and Print패턴 검색파일 안에서 글자 찾기
find(영단어) "찾다"파일 찾기파일/폴더 위치 검색
 
Terminal
$ man ls                          # ls의 전체 매뉴얼 (q로 종료)
$ which python                    # python이 설치된 위치
$ grep "error" server.log         # server.log에서 "error" 찾기
$ find . -name "*.md"             # 현재 폴더 아래 .md 파일 모두
grep 어원 원래 ed 편집기의 명령 g/re/p(global / regular expression / print = 전체에서 정규표현식으로 찾아 출력)에서 따왔습니다. 너무 유용해서 별도 명령어로 독립한 것.

2.7 기타 자주 쓰는 것

명령어약어풀이의미
echo(영단어) "메아리"출력글자 출력
clear(영단어) "비우다"화면 지우기화면 깨끗이
history(영단어) "기록"이력입력한 명령어 기록
exit(영단어) "나가다"종료터미널 종료
sudoSuperUser DO최고 권한자로 실행관리자 권한
 
Terminal
$ echo "Hello"                    # Hello
$ clear                           # 화면 깨끗이
$ history                         # 지난 명령어 목록
$ sudo apt install gh             # 관리자 권한으로 설치
$ exit                            # 터미널 닫기
⚠️
sudo 강조비밀번호를 물어보는 명령은 거의 다 sudo입니다. 시스템에 영향을 주는 명령이라는 뜻이니 신중하게 쓰세요.

3. 옵션(플래그) 약어 정리

명령어 뒤에 붙는 -r, -f, -a 같은 게 옵션(플래그)입니다. 이것도 다 영어 단어의 약어입니다.

3.1 가장 자주 만나는 옵션

옵션약어풀이의미
-r / -RRecursive재귀적으로폴더 안쪽까지
-fForce강제로확인 없이 실행
-aAll모두숨김 파일 포함
-lLong긴 형식자세히 표시
-vVerbose수다스럽게진행 상황 출력
-hHuman / Help사람이 읽기 좋게 / 도움말1024 대신 1K / 도움말
-iInteractive물어보면서작업 전 확인
-pParents / Preserve부모까지 / 유지중간 폴더 생성

3.2 합쳐 쓰기

옵션은 합쳐 쓸 수 있습니다.

Terminal
$ ls -l -a       # 같음
$ ls -la         # 이렇게도 됨
$ ls -al         # 순서 상관없음

3.3 rm -rf의 의미를 분해해보면

rm -r -f │ │ │ │ │ └─ Force : 묻지 말고 │ └────── Recursive : 폴더 안쪽까지 └─────────── Remove : 삭제하라 → "묻지 말고 폴더 안쪽 전부 지워라"

이게 왜 위험한지 보이시죠? 묻지도 않고, 폴더 통째로, 휴지통도 거치지 않습니다.

3.4 다른 예시들

Terminal
$ ls -la         # List : Long + All        → 자세히 + 숨김파일까지
$ cp -rv a b     # Copy : Recursive + Verbose → 폴더 통째 복사 + 진행상황
$ mkdir -p a/b/c # Make Directory : Parents  → 중간 폴더까지 만들기
$ rm -i file     # Remove : Interactive      → 지울 때 물어봐줘

-i 옵션은 비개발자에게 안전장치로 추천할 만합니다. 다음을 ~/.zshrc에 추가하면 rm 사용 시 항상 물어봅니다:

~/.zshrc
alias rm='rm -i'

5. Git 기본 명령어 (어원 포함)

5.1 Git이 뭔가요?

파일의 변경 이력을 저장하고, 협업할 수 있게 해주는 도구입니다.

이름 어원 만든 사람 리누스 토르발스(리눅스 창시자)가 자신을 "git"이라고 농담했는데, 영국 슬랭으로 "고집 센 영감"이라는 뜻입니다. 진지한 의미보다 농담조로 붙인 이름.

5.2 Git 용어 약어

용어풀이의미
repoREPOsitory저장소 — 프로젝트 폴더
commit(영단어) "약속하다, 맡기다"이 상태를 영구히 기록
branch(영단어) "가지"평행 작업 라인
merge(영단어) "합치다"두 가지를 하나로
HEAD(영단어) "머리"지금 보고 있는 위치
origin(영단어) "원점"원격 저장소의 기본 별명
main / master"메인" / "주"기본 브랜치 이름

5.3 처음 한 번만 하는 설정

Terminal
$ git config --global user.name "Code Villain"
$ git config --global user.email "codevillain@example.com"
config는 CONFIGuration(설정)의 약어. --global은 "이 컴퓨터 전체에 적용".

5.4 핵심 흐름: 작업 → 저장 → 공유

[작업 폴더] [GitHub] 파일 수정 │ │ git add (저장할 파일 고르기) ▼ [Staging Area] ← "무대"라는 뜻, 발표하기 전 대기 장소 │ │ git commit -m "메시지" (세이브 포인트 만들기) ▼ [로컬 저장소] │ │ git push (GitHub에 올리기) ▼ [원격 저장소]

5.5 저장소 만들기·가져오기

명령어약어/어원의미
git initINITialize "초기화"이 폴더를 Git 저장소로
git clone(영단어) "복제"원격 저장소를 통째로 받기
 
Terminal
$ git init                                              # 새 저장소 만들기
$ git clone https://github.com/villainscode/my-repo.git  # 원격 저장소 복제

5.6 상태 확인

명령어약어/어원의미
git status(영단어) "상태"지금 어떤 파일이 바뀌었나?
git log(영단어) "기록"커밋 기록 보기
git diffDIFFerence "차이"무엇이 어떻게 바뀌었나?
 
Terminal
$ git status
On branch main
Changes not staged for commit:
  modified:   README.md

$ git log --oneline    # 한 줄씩 간단히
4f1d2a3 README 첫 작성
a2b3c4d 초기 설정

$ git diff             # 줄 단위로 변경사항
💡
막혔을 때 일단 git statusGit이 친절하게 "이런 상태이고, 이렇게 하면 됩니다"라고 알려줍니다.

5.7 저장하기 (add → commit)

명령어약어/어원의미
git add(영단어) "추가"저장 후보 목록에 올리기
git commit(영단어) "약속하다"세이브 포인트 만들기
-mMessage커밋 메시지
-aAll모든 변경 자동 add
 
Terminal
$ git add README.md              # 하나만
$ git add .                      # 전부 (`.`은 "현재 폴더 전체")
$ git commit -m "README 작성"    # 커밋 + 메시지
$ git commit -am "버그 수정"     # add + commit 한 번에 (기존 파일만)

좋은 커밋 메시지

  • ✅ 로그인 버튼 색상 변경
  • ✅ 회원가입 이메일 검증 추가
  • ❌ 수정함
  • ❌ ㅇㅇ

5.8 공유하기 (push / pull)

명령어어원의미
git push(영단어) "밀다"내 변경을 원격으로 밀어 올리기
git pull(영단어) "당기다"원격 변경을 내 쪽으로 당겨오기
git fetch(영단어) "가져오다"원격 변경만 확인 (병합 안 함)
git remote(영단어) "원격"원격 저장소 설정
 
Terminal
# 처음 한 번
$ git remote add origin https://github.com/villainscode/my-blog.git
$ git push -u origin main      # -u = Upstream "기본 연결 설정"

# 이후엔
$ git push
$ git pull
origin이란?원격 저장소의 기본 별명입니다. "출신지/원점"이라는 뜻에서 옴. 다른 이름을 줘도 되지만 관례적으로 origin을 씁니다.

5.9 브랜치 (가지)

브랜치(branch) = "가지". 평행 우주에서 작업하기.

명령어어원의미
git branch"가지"브랜치 목록
git switch"전환"다른 브랜치로 이동
git switch -cCreate만들고 바로 이동
git merge"합치다"다른 브랜치를 현재로 합치기
 
Terminal
$ git branch                       # 목록
$ git switch -c feature/login      # 새 브랜치 만들고 이동
# ... 작업 후 commit ...
$ git switch main                  # 본 줄기로 복귀
$ git merge feature/login          # 작업 결과 합치기

5.10 실수 복구

명령어어원의미
git restore"복원"변경 되돌리기
git restore --staged"무대에서 내리기"git add 취소
git commit --amend"수정"직전 커밋 수정
 
Terminal
$ git restore README.md              # 변경한 내용 버리기
$ git restore --staged README.md     # add를 취소
$ git commit --amend -m "새 메시지"  # 직전 커밋 메시지 수정
⚠️
주의git reset --hard, git push --force는 데이터를 영구히 날릴 수 있습니다. 비개발자 단계에서는 피하시고, 막히면 git status 안내를 따르세요.

5.11 자주 만나는 시나리오

시나리오 1: 처음 GitHub에 올리기

Terminal

$ git push -u origin main                                   # 업스트림 설정하며 푸시

# ... 작업 ...

6. nano 편집기 사용법

6.1 nano가 뭔가요?

터미널에서 쓰는 가장 쉬운 편집기입니다. 메모장과 비슷해서 비개발자에게 가장 친절합니다.

이름 어원원래 pico 편집기의 후계자로 만들어졌고, "pico보다 더 작다"는 의미로 "nano"(나노 = 10⁻⁹, pico = 10⁻¹²보다 작은 단위)라고 이름 붙였습니다. 사실 기능은 pico보다 많은데 농담조 이름.

화면 아래에 단축키가 항상 표시되니까 외울 필요도 없습니다.

6.2 화면 보는 법

nano를 열면 화면 아래에 이런 게 보입니다.

nano 하단 메뉴
^G Help    ^O Write Out    ^W Where Is    ^K Cut
^X Exit    ^R Read File    ^\ Replace     ^U Paste

^는 Ctrl 키를 의미합니다. 즉 ^X는 Ctrl + X.

6.3 열기·저장·종료 (이것만 알면 됨)

단축키약어풀이의미
nano 파일명파일 열기 (없으면 새로 만듦)
Ctrl + OOutWrite Out — 써서 내보내기저장
Ctrl + XeXit나가다종료
Ctrl + GGet help도움 받기도움말

가장 흔한 흐름

Terminal
$ nano hello.txt
# 글자 입력...
# Ctrl + O → Enter (저장)
# Ctrl + X (종료)

6.4 편집 기능

단축키약어풀이의미
Ctrl + KKut(=Cut)잘라내기한 줄 잘라내기
Ctrl + UUncut되돌려 붙이기붙여넣기
Ctrl + WWhere is어디 있나?찾기
Ctrl + \찾아 바꾸기
Ctrl + _특정 줄 번호로 이동
Ctrl + AAhead맨 앞줄 시작으로
Ctrl + EEnd끝줄 끝으로

6.5 비개발자가 가장 자주 만나는 상황

저장하고 나갈 때 묻는 메시지

nano 종료 시
Save modified buffer?
 Y Yes    N No    ^C Cancel

→ Y 입력 후 Enter

파일 이름 확인

nano 저장 시
File Name to Write: hello.txt

→ 그대로 Enter 누르면 그 이름으로 저장

7. vi 편집기 사용법

7.1 vi가 뭔가요?

거의 모든 Unix 계열 시스템에 기본 설치되어 있는 편집기입니다.

이름 어원vi는 VIsual editor의 약어. 그 이전엔 ed(editor), ex(extended editor)가 있었고, vi는 "ex의 비주얼 모드"로 시작. 요즘은 대부분 vim이 설치되어 있는데, VI iMproved (vi 개선판)의 약어. macOS에서 vi를 치면 사실 vim이 실행됩니다.

7.2 ⚠️ 가장 먼저 외워야 할 것: 탈출법

vi에 갇혔다면:

vi 비상 탈출
Esc 키 → : 입력 → q! 입력 → Enter

읽기: "Escape, 콜론, q, 느낌표, 엔터" = 저장 안 하고 강제 종료

입력약어의미
EscESCape탈출, 빠져나오기 → 일반 모드로
:이제 명령을 칠 거예요
qQuit끝내기
!강제로! 무조건!

이것만 알아도 패닉에서 벗어날 수 있습니다.

7.3 vi의 핵심 개념: 모드(Mode)

vi는 다른 편집기와 다릅니다. 모드라는 게 있어요.

[일반 모드 (Normal)] - 화살표로 이동 - 명령어 입력 │ ┌─────┴─────┐ │ │ i 누르면 : 누르면 │ │ ▼ ▼ [입력 모드] [명령 모드] 글자 입력 :w 저장 가능 :q 종료 │ │ Esc 누르면 Enter 후 자동 복귀 │ │ └─────┬─────┘ ▼ [일반 모드]

비유 일반 모드는 "리모컨", 입력 모드는 "글쓰기 펜". 글을 쓰려면 펜으로 바꾸고(i), 다 쓰면 다시 리모컨으로 돌아갑니다(Esc).

7.4 vi 사용 흐름 (절대 외울 것)

1. vi 파일명 ← 파일 열기 2. i ← 입력 모드로 (글자 쓸 수 있음) 3. (글자 입력) 4. Esc ← 일반 모드로 복귀 5. :wq ← 저장하고 종료

이게 vi의 80%입니다.

7.5 모드 전환 (어원과 함께)

키약어풀이동작
iInsert삽입커서 앞에서 입력 시작
aAppend덧붙이기커서 뒤에서 입력 시작
oOpen line새 줄 열기아래 새 줄 만들어 입력
EscESCape탈출일반 모드로 복귀

7.6 저장·종료 (: 명령)

Esc로 일반 모드 들어간 후:

명령약어의미
:wWrite저장
:qQuit종료
:wqWrite + Quit저장하고 종료
:q!Quit + 강제저장 안 하고 강제 종료
:w 파일명다른 이름으로 저장
ZZ:wq와 같음 (shift+z 두 번)

7.7 이동 (일반 모드에서)

키어원의미
h j k l키보드 위치상 좌·하·상·우화살표 대용 (옛날엔 화살표 키가 없었음)
0(숫자 0)줄의 맨 앞
$(정규식 약속)줄의 맨 끝
ggGo to top파일 맨 위
GGo to end파일 맨 아래
숫자G그 줄 번호로 이동 (10G = 10번째 줄)
Ctrl + fForward한 페이지 아래
Ctrl + bBackward한 페이지 위
비개발자에게 hjkl 대신 그냥 화살표 키를 쓰라고 하세요. 같은 결과입니다.

7.8 편집 (일반 모드에서)

Ctrl + rRedo다시 실행

7.9 찾기 (일반 모드에서)

키어원의미

8. 한 페이지 요약

8.1 어원으로 외우는 핵심 명령어

pwd = Print Working Directory → 현재 위치 ls = LiSt → 목록 cd = Change Directory → 폴더 이동 mkdir = MaKe DIRectory → 폴더 만들기 cp = CoPy → 복사 mv = MoVe → 이동 rm = ReMove → 삭제 cat = conCATenate → 내용 출력 man = MANual → 매뉴얼 grep = Globally search RegExp Print → 패턴 검색 sudo = SuperUser DO → 관리자 권한

8.2 옵션 약어

-r/-R = Recursive → 폴더 안쪽까지 -f = Force → 묻지 말고 -a = All → 숨김 파일 포함 -l = Long → 자세히 -v = Verbose → 진행 상황 출력 -h = Human / Help → 사람이 읽기 좋게 / 도움말 -i = Interactive → 물어보면서 -p = Parents → 중간 폴더까지 -m = Message → 메시지 → rm -rf = Remove + Recursive + Force "묻지 말고 폴더 안쪽 전부 지워라" (위험!)

8.3 단축키 필수 5개

Control + C (Cancel) → 강제 종료 (막히면 일단 이거) Control + L (cLear) → 화면 깨끗이 Control + A (Ahead) → 줄 맨 앞 Control + E (End) → 줄 맨 끝 ↑ → 이전 명령어

8.4 Git 매일 쓰는 6개

git status → 뭐가 바뀌었나? git add . → 다 저장 대상으로 git commit -m "메시지" → 세이브 (-m = Message) git push → 밀어 올리기 (원격으로) git pull → 당겨오기 (원격에서) git log --oneline → 기록 보기

8.5 편집기 비상 탈출

[nano] 저장: Ctrl + O (Out) → Enter 종료: Ctrl + X (eXit) [vi] 저장 종료: Esc → :wq → Enter (Write + Quit) 저장 없이 종료: Esc → :q! → Enter (Quit + 강제)

8.6 막혔을 때 체크리스트

  1. 일단 Control + C (Cancel) 눌러봤나요?
  2. git status로 Git이 뭐라고 하는지 봤나요?
  3. vi에 갇혔으면 Esc → :q! → Enter
  4. 그래도 안 되면 터미널 창을 닫고 새로 여세요
  5. 정 안 되면 오류 메시지를 그대로 복사해서 검색하세요
