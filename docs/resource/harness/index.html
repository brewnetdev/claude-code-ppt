하네스에서 EDD까지:
CLAUDE.md 한 파일에서 정량 평가 루프까지 7강

에이전트 하네스의 최고 레버리지 포인트인 CLAUDE.md(1강)에서 출발해, 레벨을 올려가며 하네스를 완성(6강)하고, 그 위에 EDD를 얹는 방향성(7강)까지. 각 강의는 쉬운 설명 → 다이어그램 → 실행 예제 → 체크리스트 → 출처 순으로 구성했다.

이 강의가 답하는 질문

Claude Code 같은 에이전트는 "설치하면 끝"이 아니다. 공식 베스트 프랙티스의 대전제는 하나다 — 대부분의 모범 사례는 "컨텍스트 윈도우는 빨리 차고, 차면 성능이 떨어진다"는 단일 제약에서 나온다. 하네스 구축이란 결국 이 제약 아래에서 (1) 매 세션 무엇을 자동으로 알게 하고, (2) 무엇을 결정적으로 강제하고, (3) 무엇을 격리·검증할지를 계층적으로 설계하는 일이다. 근거: Claude Code 공식 Best Practices

전체 목차

  1. LV1 · 1강CLAUDE.md — 하네스 최고의 레버리지 포인트매 세션 자동 로드되는 영속 메모리. 왜 "레포에서 가장 레버리지 높은 파일"인지, 그리고 가장 흔한 오남용
  2. LV2 · 2강권한 모델과 settings.json — 프로그래머블 안전"부탁"이 아니라 "강제"가 필요한 것들. allow/ask/deny와 플랜 모드로 만드는 결정적 경계
  3. LV3 · 3강커맨드와 스킬 — 반복 워크플로의 자산화하루 두 번 이상 하는 일은 커맨드로, 조건부 지식은 스킬로. 커맨드 vs 스킬 vs 에이전트 선택 기준
  4. LV4 · 4강훅 — 요청(request)을 보증(guarantee)으로CLAUDE.md의 지시는 정중한 부탁일 뿐이다. 반드시 일어나야 하는 일은 훅에 넣는다
  5. LV5 · 5강서브에이전트와 검증 루프 — 컨텍스트 위생과 제2의 눈일한 에이전트가 스스로 채점하게 두지 않기. 격리 컨텍스트, 검증 서브에이전트, 증거 기반 완료
  6. 완성형 · 6강완성형 하네스 — research → plan → execute → review → ship레벨 1~5의 부품을 하나의 게이트 파이프라인으로. 커뮤니티 방법론들이 독립적으로 수렴한 패턴
  7. 그 이후 · 7강하네스 이후 — EDD로 가는 방향성하네스가 만든 실행·검증 능력을 정량 평가 루프로 승격시키기. 골든셋·게이트·가드레일·성숙도 LV0~5

수강 전 알아둘 것

  • 대상 도구: 예제는 Claude Code 기준이지만, "영속 컨텍스트 → 결정적 강제 → 워크플로 자산 → 자동 보증 → 격리 검증"의 레벨 구조는 모든 에이전트 하네스에 일반화된다.
  • 비유 하나로 전체 잡기: 하네스 구축은 신입 엔지니어 온보딩과 같다. LV1은 온보딩 문서, LV2는 접근 권한 정책, LV3은 사내 위키의 절차서, LV4는 CI가 강제하는 규칙, LV5는 코드 리뷰어 배정이다.
  • 근거 표기: 공식 문서(1차)와 커뮤니티 모범 사례(2차)를 구분해 표기했다. 커뮤니티발 수치(예: 컨텍스트 40% 저하 구간)는 "커뮤니티 관찰"로 명시한다.
