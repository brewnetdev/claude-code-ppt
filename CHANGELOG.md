### Added

- **Auto-linkify URLs in the editor.** Typing `https://example.com` followed
  by SPACE inside any editable text region (slide content blocks or free
  text overlays) wraps the URL in an `<a target="_blank" rel="noopener noreferrer">`
  anchor immediately, so the user gets the same blue-underline feedback
  Notion / Google Docs / Slack provide. The caret is parked in a fresh text
  node after the anchor so continued typing does not extend the link text.
  Trailing sentence punctuation (`. , ; : ! ? ) ] } >`) is left outside the
  anchor. Inside `<a>` / `<code>` / `<pre>` / `<script>` / `<style>` the
  trigger is suppressed.
  Files: `src/canvas/autoLinkUrl.ts` (new), wired into
  `src/canvas/useSlideEditing.ts` and `src/canvas/OverlayLayer.tsx`.
- **Linkify URLs at HTML / PDF export time.** The HTML bundle generated by
  `buildHtmlBundle` now wraps bare `https?://ŌĆ”` text and stamps existing
  `<a href="https://ŌĆ”">` anchors with `target="_blank" rel="noopener noreferrer"`
  before serialization. Both the standalone HTML download and the
  `window.print()`-driven PDF use the same bundle, so links open in a new
  tab in either output. Code-block and preformatted markup are skipped to
  preserve syntax-highlighted token structure.
  Files: `src/exporter/linkify.ts` (new), applied in
  `src/exporter/htmlBundle.ts`.

### Changed

- **Highlight Clear button moved next to its header.** In the text-format
  properties panel, the Clear control now sits on the right of the
  "HIGHLIGHT" label instead of after the swatch row, so it stays visible
  even when the swatches wrap onto a second line.
  Files: `src/editor/TextFormatPanel.tsx`.

### Fixed

- **Bullet-list `<li>` wrapper survives browser-default deletes.** A
  capture-phase `beforeinput` interceptor handles cross-`<li>` Backspace,
  Delete, Cut, drag-delete, word-delete, and IME deletes by replaying the
  merge through `mergeListItems`, which preserves each surviving `<li>`'s
  inner `<div>` wrapper. Single-`<li>` deletes still use the browser
  default. An `enforceBulletListInvariant` safety net runs on every
  `input` event to repair structural breakage from paste / drag / future
  handler bugs. Without these, deleting at an `<li>` boundary stripped
  the wrapper, collapsed `.bullet-list { display: flex }` and
  `.bullet-list .sub { display: block }`, and flattened the entire list
  into one inline run ("1ņŚ┤ļĪ£ ņś¼ļØ╝Ļ░ĆļŖö ļ▓äĻĘĖ").
  Files: `src/canvas/listInvariant.ts` (new),
  `src/canvas/useSlideEditing.ts` (`onListItemDelete`, `mergeListItems`,
  `onInput` safety net).
- **Image drops keep their natural resolution.** The previous
  `MAX_EDGE = 600` clamp in the drop handler forced a 6├Ś downscale on
  drops like a 3104├Ś3647 book cover, producing the "image quality tanks
  on drop" symptom. The clamp is removed; dropped images use their
  natural pixel dimensions for the overlay box and the user can shrink
  them with Moveable handles afterwards. Top-left is still clamped to Ōēź 0
  so the user always sees the corner of an oversized image.
  Files: `src/canvas/SlideCanvas.tsx` (`onDrop`).
