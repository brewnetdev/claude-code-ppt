### Added

- **Empty-row Backspace deletes the table row.**
  Pressing `Backspace` alone (no modifier, collapsed caret) inside any `<td>`
  or `<th>` cell triggers `onTableBackspace` in `useSlideEditing`. The handler
  fires only when every cell in the row has empty `textContent`; it refuses to
  drop a `<thead>` row (header rows are structural) and refuses to drop the
  last remaining `<tbody>` row (prevents a shape-less table). On deletion the
  caret parks at the end of the previous row's last cell, or ŌĆö if no previous
  row exists ŌĆö the first cell of the next row. The event listener is cleaned up
  in the return callback alongside `onTableTab`. This feature is the symmetric
  counterpart to `onTableTab`'s "Tab past the last cell appends a new row."
  File: `src/canvas/useSlideEditing.ts` (`onTableBackspace`, cleanup).

- **Slide migration system for auto-healing cached HTML.**
  `src/persistence/slideMigrations.ts` introduces a typed `Migration` interface
  and an `ALL_MIGRATIONS` array processed at deck-load time. Applied migration
  names are persisted in `localStorage` under
  `claude-code-ppt:deck-migrations:<deckId>`, so each migration runs exactly
  once per deck. The first migration (`unescape-th-inline-tags-2026-04-29`)
  surgically restores `&lt;strong&gt;` / `&lt;em&gt;` / `&lt;span&gt;` /
  `&lt;br&gt;` / `&lt;code&gt;` back to real tags inside `<th>` cells that
  were escaped by the now-fixed renderer. Users no longer need a manual Reset
  to see bold table headers ŌĆö edits are preserved.
  Files: `src/persistence/slideMigrations.ts` (new),
  `src/App.tsx` (`openDeck`).

- **Auto-linkify URLs in the editor.** Typing `https://example.com` followed
  by SPACE inside any editable text region (slide content blocks or free
  text overlays) wraps the URL in an `<a target="_blank" rel="noopener noreferrer">`
  anchor immediately, so the user gets the same blue-underline feedback
  Notion / Google Docs / Slack provide. The caret is parked in a fresh text
  node after the anchor so continued typing does not extend the link text.
  Trailing sentence punctuation (`. , ; : ! ? ) ] } >`) is left outside the
  anchor. Inside `<a>` / `<code>` / `<pre>` / `<script>` / `<style>` the
  trigger is suppressed.
  Files: `src/canvas/autoLinkUrl.ts` (new), wired into
  `src/canvas/useSlideEditing.ts` and `src/canvas/OverlayLayer.tsx`.

- **Linkify URLs at HTML / PDF export time.** The HTML bundle generated by
  `buildHtmlBundle` now wraps bare `https?://ŌĆ”` text and stamps existing
  `<a href="https://ŌĆ”">` anchors with `target="_blank" rel="noopener noreferrer"`
  before serialization. Both the standalone HTML download and the
  `window.print()`-driven PDF use the same bundle, so links open in a new
  tab in either output. Code-block and preformatted markup are skipped to
  preserve syntax-highlighted token structure.
  Files: `src/exporter/linkify.ts` (new), applied in
  `src/exporter/htmlBundle.ts`.

### Changed

- **Terminal and code-block text is brighter.**
  Base text color raised from `#cbd5e1` to `#f1f5f9`. shiki `github-dark`
  comment tokens (both legacy `#6A737D` and current `#8b949e` hex values)
  overridden via attribute selector + `!important` to `#b0b1b2`. A
  `filter: brightness(1.18) saturate(1.05)` on `pre code` lifts the entire
  token palette without enumerating every shiki inline-style rule.
  File: `src/canvas/themes/code-blocks.css`.

- **Drag handle relocated inside terminal / code-block elements.**
  `.terminal` and `.code-block` carry `overflow: hidden`, which clipped the
  reorder handle sitting at `left: -44px`. An override in `spike.css` moves
  the handle to `right: 8px; top: 8px` (inside, top-right corner) when the
  parent slot is `[data-slot="terminal"]` or `[data-slot="code"]`.
  File: `src/canvas/spike.css`.

- **`TextFormatPanel` detects current font size and color from the selection.**
  `selectionchange` now reads `getComputedStyle(anchorNode)` to populate the
  font-size input and color swatch with live values. A `lastDetectedFontSize`
  ref guards against the controlled input being overwritten mid-keystroke
  (idempotency ŌĆö setState is skipped when the detected value equals the
  current state).
  File: `src/editor/TextFormatPanel.tsx`.

- **Highlight Clear button moved next to its header.** In the text-format
  properties panel, the Clear control now sits on the right of the
  "HIGHLIGHT" label instead of after the swatch row, so it stays visible
  even when the swatches wrap onto a second line.
  Files: `src/editor/TextFormatPanel.tsx`.

### Fixed

- **`<th>` cells rendered inline tags as literal escaped text.**
  `renderComparisonTable` and `renderTable` in `src/generator/planRenderer.ts`
  used `escapeHtml` (full escape) for header cells, causing `<strong>` to
  appear as `&lt;strong&gt;` in the slide. Both call-sites (lines 222, 292)
  now use `escapeInlineHtml` (allowlist: `strong | em | span | br | code`),
  consistent with how body cells are already handled.
  File: `src/generator/planRenderer.ts`.

- **Bullet-list `<li>` wrapper survives browser-default deletes.** A
  capture-phase `beforeinput` interceptor handles cross-`<li>` Backspace,
  Delete, Cut, drag-delete, word-delete, and IME deletes by replaying the
  merge through `mergeListItems`, which preserves each surviving `<li>`'s
  inner `<div>` wrapper. Single-`<li>` deletes still use the browser
  default. An `enforceBulletListInvariant` safety net runs on every
  `input` event to repair structural breakage from paste / drag / future
  handler bugs. Without these, deleting at an `<li>` boundary stripped
  the wrapper, collapsed `.bullet-list { display: flex }` and
  `.bullet-list .sub { display: block }`, and flattened the entire list
  into one inline run ("1ņŚ┤ļĪ£ ņś¼ļØ╝Ļ░ĆļŖö ļ▓äĻĘĖ").
  Files: `src/canvas/listInvariant.ts` (new),
  `src/canvas/useSlideEditing.ts` (`onListItemDelete`, `mergeListItems`,
  `onInput` safety net).

- **Image drops keep their natural resolution.** The previous
  `MAX_EDGE = 600` clamp in the drop handler forced a 6├Ś downscale on
  drops like a 3104├Ś3647 book cover, producing the "image quality tanks
  on drop" symptom. The clamp is removed; dropped images use their
  natural pixel dimensions for the overlay box and the user can shrink
  them with Moveable handles afterwards. Top-left is still clamped to Ōēź 0
  so the user always sees the corner of an oversized image.
  Files: `src/canvas/SlideCanvas.tsx` (`onDrop`).
